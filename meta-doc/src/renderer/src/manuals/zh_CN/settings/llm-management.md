# LLM配置管理

## 概述

LLM配置管理允许您创建、编辑、删除和管理多个LLM配置。配置以**网格卡片**形式展示，类似代理客户端：每个卡片显示配置名称与类型，点击即可切换使用，支持在卡片上检查连通性，通过右键菜单进行复制、编辑、导出与删除。

## 界面布局

### 网格与卡片

1. 在LLM设置页面启用LLM后，下方会显示**配置网格**。
2. 每个**配置卡片**包含：
   - **第一行**：配置名称
   - **第二行**：大模型类型（如 OpenAI、通义千问、DeepSeek、Ollama 等）
3. **点击卡片**即可切换到该配置，当前使用的配置卡片会以**绿色边框**高亮。
4. 网格右上角有**新建配置**和**导入配置**按钮。

## 切换配置

- 在配置网格中**点击任意卡片**即可切换到该配置。
- 当前配置会以绿色边框和轻微高亮显示，所有AI功能将立即使用该配置的LLM服务。

## 检查连通性

- 每个卡片右侧有**「检查」**按钮，点击后会测试该配置的**问答流式**与**对话流式**能力。
- 测试中会显示加载图标；若有正常输出则自动停止并显示**绿色勾**；若请求错误则显示**红色叉**及简短错误信息。
- 无论结果如何，再次点击均可重新测试。

## 右键菜单

在配置卡片上**右键**可打开菜单，支持：

- **复制配置**：创建该配置的副本（新名称带「(副本)」）。
- **编辑配置**：打开编辑对话框，修改名称、类型及各项参数，**确定**即保存，**取消**则不保存。
- **导出配置**：将当前配置导出为 JSON 文件。
- **删除配置**：删除该配置（**预设配置不可删除**，见下文）。

## 预设配置

以下类型对应的**预设配置**（如「Ollama (默认)」「通义千问 (默认)」等）**不可删除**，但**允许编辑**（编辑时**不可更改大模型类型**）：

- 通义千问、DeepSeek、OpenAI官方、OpenAI兼容、Google Gemini、Ollama

自定义配置及复制得到的配置可以正常删除。

## 创建配置

### 新建配置

1. 点击网格右上角**「新建配置」**。
2. 在弹窗中输入配置名称并确认。
3. 系统会基于**当前选中的配置**创建新配置并自动切换到新配置。

**注意**：当前选中的配置为「手动」类型时，新建按钮不可用。

### 导入配置

1. 点击网格右上角**「导入配置」**。
2. 在打开的文件对话框中选择一个或多个 JSON 配置文件（支持**批量选择**）。
3. 系统会逐个读取并导入，导入的配置会追加到列表中。

支持单个配置对象或配置数组的 JSON 格式；导入时会生成新 ID，避免与现有配置冲突。

## 编辑配置

1. 在配置卡片上**右键**，选择**「编辑配置」**。
2. 在编辑对话框中修改**配置名称**、**大模型类型**（非预设配置可改）以及该类型下的各项参数（API 地址、密钥、模型等）。
3. 点击**确定**保存，或**取消**放弃修改。**无「未保存」状态**：只有确认后才会写入。

不同LLM类型的配置参数说明见 [[settings.llm-types|LLM类型配置]]。

## 删除配置

1. 在配置卡片上**右键**，选择**「删除配置」**（预设配置不显示该选项）。
2. 在确认对话框中确认删除。
3. 若删除的是当前使用的配置，系统会自动切换到其他配置。

## 导出配置

- **单个导出**：在卡片上右键 → **导出配置**，将当前配置保存为 JSON 文件。
- 导出的文件可用于备份或在其他设备/账号中通过「导入配置」恢复。

## 最佳实践

1. **命名规范**：使用清晰的配置名称，如「工作-Ollama」「实验-OpenAI」。
2. **定期备份**：重要配置定期导出备份。
3. **先检查后使用**：新配置或修改后可用卡片上的「检查」确认连通性。
4. **清理无用配置**：定期删除不再使用的配置，保持列表整洁。
