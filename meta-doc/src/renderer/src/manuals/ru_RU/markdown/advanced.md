## Диаграммы и формулы

### Блоки кода для диаграмм

В документах можно использовать блоки кода для вставки диаграмм Mermaid, PlantUML, ECharts и других. Редактор будет отображать их в реальном времени:

- **Mermaid**: блок-схемы, диаграммы последовательностей, диаграммы классов, диаграммы Ганта и др. См. [[charts.mermaid|Диаграммы Mermaid]]
- **PlantUML**: UML-диаграммы и др. См. [[charts.plantuml|Диаграммы PlantUML]]
- **ECharts**: диаграммы для визуализации данных. См. [[charts.echarts|Диаграммы ECharts]]

### Математические формулы

Поддерживаются встроенные и блочные формулы:

- **Встроенные формулы**: `$...$` или `\(...\)`
- **Блочные формулы**: `$$...$$` или `\[...\]`
- **Многострочные формулы**: используйте окружения `aligned`, `equation` и др.

### Преобразование формул LaTeX

Редактор может преобразовывать часть синтаксиса формул LaTeX в совместимую форму Markdown/HTML для корректного отображения в средах без LaTeX.

## Расширенный синтаксис

### Расширенные возможности таблиц

- **Выравнивание**: используйте `:---`, `:---:`, `---:` в строке-разделителе заголовков для установки выравнивания по левому краю, по центру и по правому краю.
- **Объединение**: сложные таблицы можно реализовать с помощью HTML `<table>`.
- **Создание из выделения**: после выделения текста в редакторе можно быстро вставить таблицу через контекстное меню или меню.

### Ссылки и изображения

- **Ссылки по ссылке**: `[текст][имя_ссылки]`, определение `[имя_ссылки]: URL` в конце документа.
- **Заголовки и атрибуты**: некоторые рендереры поддерживают `(url "заголовок")` или пользовательские атрибуты.
- **Размеры изображений**: установка ширины и высоты через HTML `<img>` или расширенный синтаксис (в зависимости от поддержки рендерера).

### Сноски

Если рендерер поддерживает расширение для сносок:

```markdown
Основной текст[^1].

[^1]: Текст сноски.
```

## Взаимодействие с функциями редактора

### Контекстное меню и ИИ

- **Оптимизация абзаца**: выделите абзац и используйте пункт «Оптимизация абзаца» в контекстном меню или функцию полировки ИИ. См. [[features.paragraph-optimization|Функция оптимизации абзацев]]
- **Вставка диаграмм**: вставка блоков кода Mermaid/ECharts и др. через контекстное меню или помощника ИИ. См. [[charts.introduction|Введение в диаграммы]]

### База знаний и автодополнение

- После включения [[knowledge-base.usage|базы знаний]] автодополнение ИИ и диалоги могут учитывать содержимое текущего документа и базы знаний.
- Настройте клавишу-триггер и максимальное количество токенов в [[ai.completion|Автодополнении ИИ]] для повышения эффективности написания длинных текстов.

## Рекомендации

1. **Сначала основы, потом расширения**: сначала хорошо освоите [[markdown.basics|базовый синтаксис]], затем используйте диаграммы и формулы.
2. **Единообразие стиля**: старайтесь использовать единообразные типы диаграмм и способы записи формул в пределах одного документа.
3. **Совместимость**: при экспорте в PDF/HTML обратите внимание на совместимость диаграмм и формул.
4. **Производительность**: слишком большое количество или размер диаграмм на одной странице может повлиять на производительность предпросмотра.
