# LLM設定

## 概要

LLM（大規模言語モデル）設定は、MetaDoc の AI 機能の中核です。**Steam 配信の製品版**では、従来のテスト系チャネルとは推奨フローが異なります。

- **Steam（推奨）**：MetaDoc が運用する **Cloudflare ベースの LLM API プロキシ**（アプリ内では **「MetaDoc Cloud (Steam)」** などの表記）を利用します。**Steam でチャージ**して **クレジット（credits）** を購入し、AI 機能を利用できます。通常は **サードパーティの API キーを自分で用意する必要はありません**。
- **自前 API（BYOK）**：**開発者向け／実験的**な扱いです。**LLM** 設定画面で **実験的オプション** を開き、**「実験的な接続を有効にする」** をオンにした場合にのみ、従来の **複数プロファイル＋カスタム API** の UI が表示されます。詳細は下記「実験的接続と BYOK」およびリンク先を参照してください。

以下では、まず **Steam / MetaDoc Cloud**（残高、チャージ、モデル切替）を説明し、次に **実験的オプション**、続いて **画面全体の操作**（LLM の有効化、温度、設定グリッドなど）を説明します。

---

## Steam：MetaDoc Cloud とクレジット（推奨）

**Steam** からインストール・起動する MetaDoc 向けです。

### 手順の概要

1. **LLM 設定を開く**：**設定** → **LLM**（またはメニューバーから **設定**）。
2. **「MetaDoc Cloud (Steam)」** を探す：**残高（credits）**、**モデル** の選択、**チャージ／クレジット追加**、サーバー側との **更新（リフレッシュ）** が行えます。
3. **チャージ**：**チャージ** から **Steam のアプリ内課金** の流れで credits を購入します。Steam ライブラリから起動し、Steam クライアントが利用可能である必要があります（購入できない場合は、UI に Steam／Greenworks などの条件が表示されます）。
4. **残高確認**：このページに credits が表示されます。トレイの **Steam** から **アカウントの credits 残高** を確認しチャージを始められます（文言はアプリ内と一致します）。
5. **料金**：**モデル** ごとに credits の消費ルールが異なる場合があります。詳細はアプリの表示に従ってください。
6. **モデル切替**：MetaDoc Cloud で **モデル** を選ぶと、**AI チャット、補完、校正、Agent** などはそのモデルで呼び出されます。利用中にいつでも切り替え可能です。

### 補足

- エンドポイントとプロキシは MetaDoc 側で運用されるため、この経路では OpenAI／DeepSeek などの **URL とキーを貼らなくても** 利用できます（下記 BYOK とは別です）。
- 残高不足やネットワーク障害時は AI 機能が使えない／エラーになることがあります。**チャージ** または **残高の更新** を試してください。

## 開発者向け：実験的接続と BYOK（自前 API）

**Steam** 版では、従来の **「自分で LLM API を設定する」** フローは **開発者向け／実験的** です。**LLM** 設定の **実験的オプション** を開き、**「実験的な接続を有効にする」** をオン（確認ダイアログあり）にすると、**複数設定カード、API ベース URL、API キー、プロバイダ種別** など、旧テストビルドと同様の画面が表示されます。

**重要**：

- **実験的**であり、標準の Steam／MetaDoc Cloud 体験と **異なる** 場合があります。**サードパーティに直接課金** したり、ネットワーク・コンプライアンス上のリスクが増えたりします。
- **自前 API 利用に伴うキー管理、課金、可用性、利用規約はすべてユーザー責任** です。MetaDoc はクライアント側の設定と接続機能を提供するのみです。

### AI機能の有効化

LLM設定ページでは、まずLLM機能を有効にする必要があります：

1. 「LLMを有効にする」スイッチを見つける
2. スイッチを「有効」状態に切り替える
3. システムは自動的にデフォルトのLLM設定を読み込みます

上図は、LLM設定インターフェースの主要な構成要素を示しています：

- **グローバル設定**：LLM有効化スイッチ、温度（Temperature）調整スライダー、thinkタグ削除オプション、ターミナル実行デフォルト許可など
- **設定グリッド**：すべての設定をカード形式で表示。各カードには設定名とタイプ（例：OpenAI、通義千問、DeepSeek、Ollamaなど）が表示されます。カードをクリックすると使用する設定を切り替えられ、現在の設定は緑色の枠でハイライト表示されます
- **カード操作**：カードの右側で、その設定のQ&Aおよび対話ストリーミング能力を「チェック」できます。右クリックメニュでは、コピー、編集、エクスポート、削除がサポートされています
- **上部操作**：グリッドの右上で、新しい設定を作成したり、ファイルから設定を一括インポートしたりできます

デモモードでは、インターフェースレイアウトを対話的に確認できますが、変更は実際には保存されません。

LLMを有効にした後、以下のAI機能を使用できます：

- AI対話
- AI校正
- AI自動補完
- AIアシスタント機能
- Agentフレームワーク

**注意事項**：

- LLMを有効にすると、一部の機能がAPIを呼び出し、費用が発生する可能性があります
- 先にLLMサービスを設定してから有効にすることをおすすめします
- AI機能が不要な場合は、リソース節約のためオフにできます

### 温度パラメータの理解

温度（Temperature）は、AIが生成するテキストのランダム性を制御するパラメータです：

- **低温度（0-0.5）**：結果はより決定的で一貫性があり、正確な回答が必要な場面に適しています
- **中温度（0.5-1.0）**：創造性と正確性のバランスが取れ、ほとんどの場面に適しています
- **高温度（1.0-2.0）**：結果はより多様で創造的になり、創作に適しています

### 設定の目安

- **技術文書**：0.3-0.5を推奨（内容の正確さを確保）
- **創作**：0.7-1.0を推奨（多様性を高める）
- **コード生成**：0.2-0.4を推奨（コードの正確さを確保）
- **会話**：0.7-0.9を推奨（自然な対話を維持）

温度設定は、AIチャット、補完、校正など、LLMを使用するすべての機能に影響します。

## 推論タグの自動削除

### 機能説明

一部のLLMは、生成内容に推論過程（思考過程）を含めることがあり、特別なタグでマークされることがあります。「推論タグを自動削除」を有効にすると、MetaDocはこれらのタグを自動的にフィルタし、最終的に生成された内容のみを残します。

**適用例**：

- 推論過程をサポートするLLMを使用する場合（例：一部のオープンソースモデル）
- 出力をより簡潔にしたい場合
- AIの思考過程を表示する必要がない場合

**注意**：

- LLMが推論タグをサポートしていない場合、このオプションは効果がありません
- 場合によっては、推論過程を残すことでAIの判断根拠の理解に役立ちます

## 設定管理

<SettingLlmSection mode="demo" />

### 複数設定のサポート

MetaDocは複数のLLM設定を作成でき、異なるシーンで異なるモデルを使い分けられます：

- **業務用設定**：日々の作業向けの安定したモデル
- **実験用設定**：新モデルや新機能のテスト
- **異なるプロバイダ**：各LLMサービスごとに独立した設定

### 設定の切り替え

LLM設定ページの設定グリッドでは：

1. **設定を選択**：任意のカードをクリックしてその設定に切り替え
2. **設定情報の確認**：各カードに設定名とタイプが表示
3. **現在使用中の設定**：緑枠でハイライト

切り替え後、すべてのAI機能は新しい設定のLLMを即座に使用します。編集はカードの右クリックメニューから「設定を編集」でダイアログを開き、OKで保存、キャンセルで破棄します。

## 注意事項

1. **APIキーの安全**：APIキーは厳重に管理し、他人と共有しないでください（BYOK／実験的接続を使う場合に該当）
2. **コスト**：LLM利用には費用がかかる場合があります。MetaDoc Cloud は **credits**、BYOK は各プロバイダの課金に従います
3. **ネットワーク**：外部APIには安定した接続が必要です
4. **バックアップ**：重要な設定はエクスポートしてバックアップしてください
5. **モデル選択**：モデルごとに能力と制限が異なります
