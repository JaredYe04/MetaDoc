# LLM 설정

## 개요

LLM(대형 언어 모델) 설정은 MetaDoc AI 기능의 핵심 설정입니다. LLM을 구성함으로써 AI 대화, AI 교정, AI 자동 완성 등 지능형 기능을 활성화할 수 있습니다. MetaDoc은 다양한 LLM 서비스 제공업체를 지원하며, 필요에 따라 적절한 모델을 선택할 수 있습니다.

### AI 기능 켜기

LLM 설정 페이지에서 먼저 LLM 기능을 활성화해야 합니다:

1. "LLM 활성화" 스위치를 찾습니다.
2. 스위치를 "활성화" 상태로 전환합니다.
3. 시스템이 기본 LLM 구성을 자동으로 로드합니다.

위 그림은 LLM 설정 인터페이스의 주요 구성 요소를 보여줍니다:

- **전역 설정**: LLM 활성화 스위치, 온도(Temperature) 조절 슬라이더, think 태그 제거 옵션, 터미널 실행 기본 허용 등
- **구성 그리드**: 모든 구성을 카드 형태로 표시하며, 각 카드는 구성 이름과 유형(예: OpenAI, 통이천문, DeepSeek, Ollama 등)을 표시합니다. 카드를 클릭하면 사용할 수 있으며, 현재 구성은 녹색 테두리로 강조 표시됩니다.
- **카드 작업**: 카드 오른쪽에서 해당 구성의 질의응답 및 대화 스트리밍 능력을 '확인'할 수 있습니다. 오른쪽 클릭 메뉴를 통해 복사, 편집, 내보내기, 삭제를 지원합니다.
- **상단 작업**: 그리드 오른쪽 상단에서 새 구성을 만들거나 파일에서 구성을 일괄 가져올 수 있습니다.

데모 모드에서는 인터페이스 레이아웃을 대화형으로 볼 수 있지만, 수정 사항은 실제로 저장되지 않습니다.

LLM을 활성화한 후에야 다음 AI 기능을 사용할 수 있습니다:

- AI 대화
- AI 교정
- AI 자동 완성
- AI 어시스턴트 기능
- Agent 프레임워크

**주의사항**:

- LLM을 활성화하면 일부 기능이 API를 호출하여 비용이 발생할 수 있습니다.
- AI 기능을 사용하기 전에 LLM 서비스를 먼저 구성하는 것이 좋습니다.
- AI 기능이 필요하지 않다면 리소스를 절약하기 위해 비활성화 상태를 유지할 수 있습니다.

## LLM 온도 설정

<SettingLlmSection mode="demo" />

### 온도 매개변수 이해하기

온도(Temperature)는 AI가 생성하는 텍스트의 무작위성을 제어하는 매개변수입니다:

- **낮은 온도 (0-0.5)**: 생성 결과가 더 결정적이고 일관적이며, 정확한 답변이 필요한 시나리오에 적합합니다.
- **중간 온도 (0.5-1.0)**: 창의성과 정확성의 균형을 맞추며, 대부분의 시나리오에 적합합니다.
- **높은 온도 (1.0-2.0)**: 생성 결과가 더 다양하고 창의적이며, 창의적인 글쓰기에 적합합니다.

### 설정 권장사항

- **기술 문서**: 0.3-0.5 권장, 내용의 정확성을 보장합니다.
- **창의적 글쓰기**: 0.7-1.0 권장, 내용의 다양성을 높입니다.
- **코드 생성**: 0.2-0.4 권장, 코드의 정확성을 보장합니다.
- **대화 교류**: 0.7-0.9 권장, 대화를 자연스럽고 유창하게 유지합니다.

온도 설정은 AI 대화, AI 완성, AI 교정 등 LLM을 사용하는 모든 기능에 영향을 미칩니다.

## 자동 추론 태그 제거

### 기능 설명

일부 LLM은 콘텐츠를 생성할 때 추론 과정(thinking process)을 포함할 수 있으며, 이 내용은 일반적으로 특수 태그로 표시됩니다. "자동 추론 태그 제거"를 활성화하면 MetaDoc은 이러한 태그를 자동으로 걸러내고 최종 생성된 콘텐츠만 유지합니다.

**적용 시나리오**:

- 추론 과정을 지원하는 LLM(예: 일부 오픈소스 모델)을 사용하는 경우
- 출력 결과를 더 간결하게 원하는 경우
- AI의 사고 과정을 볼 필요가 없는 경우

**주의사항**:

- 사용 중인 LLM이 추론 태그를 지원하지 않으면 이 옵션은 영향을 미치지 않습니다.
- 경우에 따라 추론 과정을 유지하는 것이 AI의 의사 결정 논리를 이해하는 데 도움이 될 수 있습니다.

## 구성 관리

<SettingLlmSection mode="demo" />

### 다중 구성 지원

MetaDoc은 여러 LLM 구성을 생성할 수 있어 다양한 시나리오에서 다른 모델을 사용하기 편리합니다:

- **작업 구성**: 일상 업무용으로 안정적이고 신뢰할 수 있는 모델 사용
- **실험 구성**: 새 모델이나 새 기능을 테스트하는 데 사용
- **다른 제공업체**: 다른 LLM 서비스에 대해 독립적인 구성 생성

### 구성 전환

LLM 설정 페이지의 구성 그리드에서 다음을 수행할 수 있습니다:

1. **구성 선택**: 아무 구성 카드를 클릭하면 해당 구성으로 전환됩니다.
2. **구성 정보 보기**: 각 카드는 구성 이름과 유형을 표시합니다.
3. **현재 구성 식별**: 현재 사용 중인 구성 카드는 녹색 테두리로 강조 표시됩니다.

구성을 전환하면 모든 AI 기능이 즉시 새 구성의 LLM 서비스를 사용합니다. 구성을 편집하려면 카드 오른쪽 클릭 메뉴의 '구성 편집'을 통해 대화 상자를 열고, 대화 상자에서 수정한 후 '확인'을 클릭하여 저장하거나 '취소'를 클릭하여 저장하지 않습니다. 인터페이스는 더 이상 '저장되지 않음' 상태를 구분하지 않습니다.

## 주의사항

1. **API 키 보안**: API 키를 안전하게 보관하고 타인과 공유하지 마십시오.
2. **비용 관리**: LLM 서비스 사용 시 비용이 발생할 수 있으므로 사용량에 유의하십시오.
3. **네트워크 연결**: 외부 API 사용을 위해서는 안정적인 네트워크 연결이 필요합니다.
4. **구성 백업**: 중요한 구성은 분실을 방지하기 위해 내보내기 백업을 권장합니다.
5. **모델 선택**: 모델마다 다른 능력과 제한이 있으므로 필요에 따라 선택하십시오.
