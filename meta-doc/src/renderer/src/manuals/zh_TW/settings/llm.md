# LLM 設定

## 概述

LLM（大型語言模型）設定是 MetaDoc AI 功能的核心。**透過 Steam 取得的正式版本**與早期測試管道在預設建議路徑上有所不同：

- **Steam 管道（建議）**：使用我們託管的、**基於 Cloudflare 的 LLM API 代理服務**（介面中顯示為 **「MetaDoc 官方雲（Steam）」**），透過 **Steam 儲值** 取得 **credits** 額度後使用 AI，一般**無需**自行申請第三方 API Key。
- **自備 API（BYOK）**：屬於 **開發人員選項**：僅在 **LLM 設定** 中開啟 **實驗性選項** 並啟用 **「啟用實驗性連線」** 後，才會顯示與原先測試管道類似的「多設定＋自訂 API」流程；詳見下文「開發人員選項與自備 API」及連結文件。

下文先說明 **Steam 官方雲** 的儲值、餘額與模型切換，再說明 **實驗性選項** 與 **完整介面功能**（啟用 LLM、溫度、設定網格等）。

---

## Steam 管道：MetaDoc 官方雲與額度（建議）

適用於透過 **Steam** 安裝、執行的 MetaDoc。

### 使用方式概要

1. **開啟 LLM 設定**：**設定** → **LLM 設定**（或透過頂部選單進入 **設定**）。
2. **找到「MetaDoc 官方雲（Steam）」**：在該區域可檢視 **剩餘額度（credits）**、選擇 **模型**、進行 **儲值**，以及使用 **重新整理** 同步雲端餘額。
3. **儲值**：點選 **儲值**，依 Steam **應用程式內購（微交易）** 流程購買 credits；請從 Steam 收藏庫正常啟動本應用程式，並確保 Steam 用戶端可用（若無法發起購買，介面會提示檢查 Steam／Greenworks 等條件）。
4. **檢視餘額**：頁面內顯示目前 **credits**；也可透過系統匣中的 **Steam** 入口檢視 **帳戶 Credits 餘額** 並發起儲值（與介面文案一致）。
5. **計價**：不同 **模型** 的消耗規則可能不同，扣費會依各模型對應的 credits 計價從餘額中扣除；具體以應用程式內顯示為準。
6. **隨時切換模型**：在官方雲區域選擇不同 **模型** 後，**AI 對話、補全、校對、Agent 等** 功能會依目前所選模型呼叫服務；您可在使用過程中依需求切換。

### 說明

- 此路徑由官方維護伺服端與代理，**您無需**自行填寫 OpenAI／DeepSeek 等第三方 API 位址與金鑰即可使用（與下方「自備 API」相對）。
- 若餘額不足或網路異常，相關 AI 功能可能無法使用或會顯示錯誤，請先 **儲值** 或 **重新整理額度** 後再試。

## 開發人員選項：實驗性連線與自備 API（BYOK）

在 **Steam 管道** 中，原先面向測試管道的 **「自己設定 LLM API」**（BYOK）能力放在 **開發人員選項** 中：需先在 **LLM 設定** 頁面展開 **「實驗性選項」**，再啟用 **「啟用實驗性連線」**（啟用前會有確認提示）。啟用後，才會顯示 **多設定卡片、API 位址、API Key、多提供商類型** 等與歷史版本一致的設定介面。

**重要**：

- 此選項為 **實驗性質**，可能與正式、簡化的 Steam 官方雲體驗 **不一致**，並可能導致 **直接向第三方服務商付費**、網路與合規等額外風險。
- **因自備 API 產生的金鑰保管、費用、可用性及與第三方服務條款相關的問題，均由您自行承擔**；MetaDoc 僅提供用戶端側的設定與連線能力。

## LLM 溫度設定

<SettingLlmSection mode="demo" />

### 理解溫度參數

溫度（Temperature）是控制 AI 產生文字隨機性的參數：

- **低溫度（0-0.5）**：產生結果較具確定性與一致性，適合需要準確答案的情境
- **中等溫度（0.5-1.0）**：平衡創造性與準確性，適合大多數情境
- **高溫度（1.0-2.0）**：產生結果較多樣、具創意，適合創意寫作

### 設定建議

- **技術文件**：建議 0.3-0.5，確保內容準確
- **創意寫作**：建議 0.7-1.0，增加內容多樣性
- **程式碼產生**：建議 0.2-0.4，保證程式碼準確性
- **對話交流**：建議 0.7-0.9，維持對話自然流暢

溫度設定會影響所有使用 LLM 的功能，包括 AI 對話、AI 補全、AI 校對等。

## 自動移除推理標籤

### 功能說明

某些 LLM 在產生內容時可能會包含推理過程（thinking process），這些內容通常以特殊標籤標記。啟用「自動移除推理標籤」後，MetaDoc 會自動過濾這些標籤，只保留最終產生的內容。

**適用情境**：

- 使用支援推理過程的 LLM（如某些開源模型）
- 希望輸出結果更簡潔
- 不需要檢視 AI 的思考過程

**注意事項**：

- 若您的 LLM 不支援推理標籤，此選項不會產生影響
- 某些情況下，保留推理過程可能有助於理解 AI 的決策邏輯

## 設定管理

<SettingLlmSection mode="demo" />

### 多設定支援

MetaDoc 支援建立多個 LLM 設定，方便您在不同情境下使用不同的模型：

- **工作設定**：用於日常工作，使用穩定可靠的模型
- **實驗設定**：用於測試新模型或新功能
- **不同提供商**：為不同的 LLM 服務建立獨立設定

### 切換設定

在 LLM 設定頁面的設定網格中，您可以：

1. **選擇設定**：點選任意設定卡片即可切換到該設定
2. **檢視設定資訊**：每張卡片顯示設定名稱與類型
3. **辨識目前設定**：目前使用的設定卡片會以綠色邊框醒目提示

切換設定後，所有 AI 功能會立即使用新設定的 LLM 服務。編輯設定需透過卡片右鍵選單中的「編輯設定」開啟對話框，在對話框中修改後點選「確定」儲存，點選「取消」則不儲存；介面不再區分「未儲存」狀態。

## 注意事項

1. **API 金鑰安全**：請妥善保管您的 API 金鑰，不要與他人分享（僅在使用自備 API／實驗性連線時適用）
2. **費用控制**：使用 LLM 服務可能會產生費用，請注意使用量；Steam 官方雲以 credits 為準，自備 API 以第三方服務商計費為準
3. **網路連線**：使用外部 API 需要穩定的網路連線
4. **設定備份**：重要設定建議匯出備份，避免遺失
5. **模型選擇**：不同模型有不同的能力與限制，請依需求選擇
