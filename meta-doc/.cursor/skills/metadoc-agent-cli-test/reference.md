# Agent CLI 参考

## 单步与进度

- **单步执行**：CLI 模式下每发一条消息，只执行「一步」（一次 LLM 回复或一次工具调用后即停），不自动连续多轮。便于逐步看结果、再发「继续」或下一条指令。
- **进度到 stdout**：执行中的阶段（如 `thinking: ...`、`tool-calling: 工具调用完成`）会实时写到 TCP，客户端将带 `[p] ` 前缀的行打印到终端，最终一行无前缀为助手回复。

## 端口与环境变量

| 变量                | 默认值 | 说明                                          |
| ------------------- | ------ | --------------------------------------------- |
| `AGENT_CLI_PORT`    | 49384  | TCP 端口，与主进程 `--agent-cli-port` 一致    |
| `AGENT_CLI_WAIT_MS` | 60000  | send-and-receive 等端口就绪的最长时间（毫秒） |

## CRUD 测试用提示词（可复制）

以下提示词针对工作区根目录，可直接用于 `send-and-receive.mjs` 或交互式 CLI。

- **列出根目录**：`列出当前工作区根目录下的所有文件和文件夹。`
- **创建文件夹**：`在当前工作区根目录下创建一个文件夹，名为 agent-test-dir。`
- **创建文件（中文）**：`在工作区根目录下创建一个文件 agent-test.txt，内容为：你好世界，这是 agent 写入的中文。`
- **读取文件**：`读取工作区根目录下的 agent-test.txt 并告诉我文件内容。`
- **修改文件**：`把工作区根目录下的 agent-test.txt 的内容改成：已修改。`
- **删除**：`删除工作区根目录下的 agent-test.txt。` / `删除工作区根目录下的 agent-test-dir 文件夹。`

## 验证要点

- **List**：返回中应包含工作区路径下的真实子项。
- **Create dir**：用资源管理器或 `dir` 确认文件夹存在。
- **Create file + 中文**：文件存在且打开后内容含“你好世界”等中文。
- **Read**：agent 回复中的内容与文件内容一致。
- **Update/Delete**：修改后再次读取、删除后列表/读取应体现变化。

## npm 脚本（package.json）

- `agent-cli`：交互式 CLI 客户端
- `agent-cli:quiet`：同上，减少 stderr 输出
- `agent-cli:dev`：开发模式启动 MetaDoc 并开启 agent-cli（端口 49384）
- `agent-cli:test`：非交互发送两条简单消息并收回复（连通性快速验证）
