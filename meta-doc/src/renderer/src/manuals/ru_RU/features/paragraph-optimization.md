# Функция оптимизации абзацев

## Обзор

Функция оптимизации абзацев позволяет использовать ИИ для оптимизации конкретных абзацев или разделов в документе. Вы можете открыть функцию оптимизации абзацев из контекстного меню или из представления структуры, чтобы сгенерировать или оптимизировать содержимое абзаца.

## Открытие оптимизации абзацев

### Открытие из контекстного меню

В редакторе можно открыть оптимизацию абзацев через контекстное меню:

1.  **Выделите текст**: Выделите текст, который нужно оптимизировать, в редакторе.
2.  **Контекстное меню**: Щелкните правой кнопкой мыши по выделенному тексту.
3.  **Выберите оптимизацию**: В контекстном меню выберите "Оптимизация абзаца" или аналогичный пункт.
4.  **Открытие диалога**: Откроется диалоговое окно оптимизации абзацев.

### Открытие из структуры

В представлении структуры можно открыть оптимизацию абзацев:

1.  **Выберите узел**: Выберите узел, который нужно оптимизировать, в дереве структуры.
2.  **Контекстное меню**: Щелкните правой кнопкой мыши по узлу.
3.  **Выберите оптимизацию**: В контекстном меню выберите "Оптимизация абзаца" или аналогичный пункт.
4.  **Открытие диалога**: Откроется диалоговое окно оптимизации абзацев.

### Автоматическое определение раздела

Оптимизация абзацев автоматически определяет текущий раздел:

-   **Позиция курсора**: Определяет текущий раздел на основе позиции курсора.
-   **Выделенный текст**: Если текст выделен, используется выделенный текст.
-   **Узел структуры**: Если открыто из структуры, используется соответствующий узел структуры.

## Параметры оптимизации

### Режим оптимизации

Можно выбрать различные режимы оптимизации:

-   **Сгенерировать содержимое**: Создать новое содержимое абзаца.
-   **Оптимизировать содержимое**: Оптимизировать существующее содержимое абзаца.
-   **Добавить содержимое**: Добавить новое содержимое после существующего.
-   **Заменить содержимое**: Заменить существующее содержимое абзаца.

### Режим контекста

Можно выбрать режим контекста:

-   **Контекст всего документа**: Использовать весь документ в качестве контекста.
-   **Контекст раздела**: Использовать только текущий раздел в качестве контекста.
-   **Без контекста**: Не использовать информацию о контексте.

### Пользовательский промпт

Можно ввести пользовательский промпт:

-   **Цель оптимизации**: Описать цель оптимизации.
-   **Требования к содержимому**: Указать требования к содержимому.
-   **Требования к стилю**: Определить стиль написания.

### Предустановленные промпты

Можно использовать предустановленные промпты:

-   **Расширить содержимое**: Расширить содержимое абзаца.
-   **Сократить содержимое**: Сократить содержимое абзаца.
-   **Переписать содержимое**: Переписать содержимое абзаца.
-   **Дополнить содержимое**: Дополнить содержимое абзаца.

## Генерация содержимого

### Процесс генерации

Процесс генерации содержимого:

1.  **Анализ раздела**: Анализировать структуру и содержимое текущего раздела.
2.  **Построение промпта**: Построить промпт для оптимизации на основе выбранных параметров.
3.  **Вызов ИИ**: Вызвать ИИ для генерации оптимизированного содержимого.
4.  **Отображение результата**: Показать сгенерированное содержимое в диалоговом окне.

### Результат генерации

Сгенерированное содержимое отображается в диалоговом окне:

-   **Предпросмотр содержимого**: Можно предварительно просмотреть сгенерированное содержимое.
-   **Редактирование содержимого**: Можно редактировать сгенерированное содержимое.
-   **Применение содержимого**: Можно применить содержимое к документу.

### Параметры генерации

При генерации можно настроить параметры:

-   **Потоковый вывод**: Отображать процесс генерации в реальном времени.
-   **Единовременная генерация**: Показать результат после завершения генерации.
-   **Отмена генерации**: Можно в любой момент отменить процесс генерации.

## Применение содержимого

### Способ применения

Сгенерированное содержимое можно применить к документу:

-   **Заменить**: Заменить исходное содержимое абзаца.
-   **Вставить**: Вставить содержимое в указанную позицию.
-   **Добавить**: Добавить содержимое в конец абзаца.

### Место применения

Можно указать место применения:

-   **Текущая позиция**: Применить в текущей позиции курсора.
-   **Позиция раздела**: Применить в начале раздела.
-   **Конец раздела**: Применить в конце раздела.

## Функция диалога

### Продолжение диалога

После генерации содержимого можно продолжить диалог:

1.  **Открыть диалог**: Нажмите кнопку "Продолжить диалог".
2.  **Перейти в диалог**: Перейти в интерфейс диалога с ИИ.
3.  **Продолжить оптимизацию**: Можно продолжить оптимизацию или изменение содержимого.

### Контекст диалога

Диалог будет включать следующий контекст:

-   **Исходное содержимое**: Исходное содержимое абзаца.
-   **Сгенерированное содержимое**: Сгенерированное содержимое.
-   **История оптимизации**: История оптимизации.

## Рекомендации

1.  **Определите цель**: Четко определите цель оптимизации, используйте ясные промпты.
2.  **Выберите контекст**: В зависимости от ситуации выберите подходящий режим контекста.
3.  **Предварительный просмотр**: После генерации просмотрите содержимое, чтобы убедиться, что оно соответствует требованиям.
4.  **Редактирование и корректировка**: После генерации можно дополнительно отредактировать и скорректировать содержимое.
5.  **Многократная оптимизация**: Можно выполнять оптимизацию несколько раз, постепенно совершенствуя содержимое.

## Важные замечания

1.  **Определение раздела**: Убедитесь, что раздел определен правильно, чтобы избежать оптимизации не того содержимого.
2.  **Использование контекста**: Разумно используйте контекст, избегайте слишком длинного содержимого.
3.  **Качество содержимого**: Сгенерированное содержимое требует ручной проверки и корректировки.
4.  **Расход токенов**: Функция оптимизации расходует токены, следите за объемом использования.
5.  **Сохранение документа**: После применения содержимого не забудьте сохранить документ.
