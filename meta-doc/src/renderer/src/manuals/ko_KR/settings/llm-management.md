# LLM 설정 관리

## 개요

LLM 설정 관리를 통해 여러 LLM 구성을 생성, 편집, 삭제 및 관리할 수 있습니다. 구성은 **그리드 카드** 형태로 표시되며, 에이전트 클라이언트와 유사합니다: 각 카드는 구성 이름과 유형을 표시하며, 클릭하여 사용할 수 있고, 카드에서 연결성을 확인할 수 있으며, 우클릭 메뉴를 통해 복사, 편집, 내보내기 및 삭제를 지원합니다.

## 인터페이스 레이아웃

### 그리드와 카드

1. LLM 설정 페이지에서 LLM을 활성화하면 하단에 **구성 그리드**가 표시됩니다.
2. 각 **구성 카드**에는 다음이 포함됩니다:
   - **첫 번째 줄**: 구성 이름
   - **두 번째 줄**: 대형 언어 모델 유형 (예: OpenAI, 통이천문, DeepSeek, Ollama 등)
3. **카드를 클릭**하면 해당 구성으로 전환되며, 현재 사용 중인 구성 카드는 **녹색 테두리**로 강조 표시됩니다.
4. 그리드 오른쪽 상단에는 **새 구성 만들기**와 **구성 가져오기** 버튼이 있습니다.

## 구성 전환

- 구성 그리드에서 **아무 카드나 클릭**하여 해당 구성으로 전환할 수 있습니다.
- 현재 구성은 녹색 테두리와 약간의 강조 표시로 나타나며, 모든 AI 기능은 즉시 해당 구성의 LLM 서비스를 사용합니다.

## 연결성 확인

- 각 카드 오른쪽에는 **「확인」** 버튼이 있습니다. 클릭하면 해당 구성의 **질의응답 스트리밍** 및 **대화 스트리밍** 능력을 테스트합니다.
- 테스트 중에는 로딩 아이콘이 표시됩니다; 정상적인 출력이 있으면 자동으로 중지되고 **녹색 체크 표시**가 나타납니다; 요청 오류가 있으면 **빨간색 엑스 표시**와 간단한 오류 메시지가 표시됩니다.
- 결과에 관계없이 다시 클릭하면 재테스트할 수 있습니다.

## 우클릭 메뉴

구성 카드에서 **우클릭**하면 메뉴가 열리며, 다음을 지원합니다:

- **구성 복사**: 해당 구성의 사본을 생성합니다 (새 이름에 「(사본)」이 붙음).
- **구성 편집**: 편집 대화 상자를 열어 이름, 유형 및 각종 매개변수를 수정합니다. **확인**을 클릭하면 저장되고, **취소**를 클릭하면 저장되지 않습니다.
- **구성 내보내기**: 현재 구성을 JSON 파일로 내보냅니다.
- **구성 삭제**: 해당 구성을 삭제합니다 (**사전 설정된 구성은 삭제할 수 없음**, 아래 참조).

## 사전 설정된 구성

다음 유형에 해당하는 **사전 설정된 구성** (예: 「Ollama (기본)」「통이천문 (기본)」 등)은 **삭제할 수 없지만**, **편집은 허용됩니다** (편집 시 **대형 언어 모델 유형은 변경할 수 없음**):

- 통이천문, DeepSeek, OpenAI 공식, OpenAI 호환, Google Gemini, Ollama

사용자 정의 구성 및 복사로 얻은 구성은 정상적으로 삭제할 수 있습니다.

## 구성 생성

### 새 구성 만들기

1. 그리드 오른쪽 상단의 **「새 구성 만들기」**를 클릭합니다.
2. 팝업 창에서 구성 이름을 입력하고 확인합니다.
3. 시스템은 **현재 선택된 구성**을 기반으로 새 구성을 생성하고 자동으로 새 구성으로 전환합니다.

**참고**: 현재 선택된 구성이 「수동」 유형일 경우, 새로 만들기 버튼을 사용할 수 없습니다.

### 구성 가져오기

1. 그리드 오른쪽 상단의 **「구성 가져오기」**를 클릭합니다.
2. 열리는 파일 대화 상자에서 하나 이상의 JSON 구성 파일을 선택합니다 (**일괄 선택** 지원).
3. 시스템은 하나씩 읽어 들여 가져오며, 가져온 구성은 목록에 추가됩니다.

단일 구성 객체 또는 구성 배열의 JSON 형식을 지원합니다; 가져올 때 새 ID가 생성되어 기존 구성과의 충돌을 방지합니다.

## 구성 편집

1. 구성 카드에서 **우클릭**하고 **「구성 편집」**을 선택합니다.
2. 편집 대화 상자에서 **구성 이름**, **대형 언어 모델 유형** (사전 설정된 구성이 아닐 경우 변경 가능) 및 해당 유형의 각종 매개변수 (API 주소, 키, 모델 등)를 수정합니다.
3. **확인**을 클릭하여 저장하거나 **취소**를 클릭하여 수정을 포기합니다. **「저장되지 않음」 상태 없음**: 확인을 클릭한 후에만 기록됩니다.

다른 LLM 유형의 구성 매개변수 설명은 [[settings.llm-types|LLM 유형 구성]]을 참조하십시오.

## 구성 삭제

1. 구성 카드에서 **우클릭**하고 **「구성 삭제」**를 선택합니다 (사전 설정된 구성에는 이 옵션이 표시되지 않음).
2. 확인 대화 상자에서 삭제를 확인합니다.
3. 현재 사용 중인 구성을 삭제하는 경우, 시스템은 자동으로 다른 구성으로 전환됩니다.

## 구성 내보내기

- **단일 내보내기**: 카드에서 우클릭 → **구성 내보내기**를 선택하여 현재 구성을 JSON 파일로 저장합니다.
- 내보낸 파일은 백업 또는 다른 장치/계정에서 「구성 가져오기」를 통해 복원하는 데 사용할 수 있습니다.

## 모범 사례

1. **명명 규칙**: 「작업-Ollama」「실험-OpenAI」와 같이 명확한 구성 이름을 사용하십시오.
2. **정기 백업**: 중요한 구성은 정기적으로 내보내 백업하십시오.
3. **사용 전 확인**: 새 구성이나 수정 후에는 카드의 「확인」을 사용하여 연결성을 확인하십시오.
4. **불필요한 구성 정리**: 더 이상 사용하지 않는 구성을 정기적으로 삭제하여 목록을 깔끔하게 유지하십시오.
