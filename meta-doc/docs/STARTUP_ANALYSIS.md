## 详细启动步骤分析

### 阶段 1: Electron 主进程初始化

**文件**: `meta-doc/src/main/index.ts`

**关键操作**:
1. **环境变量加载** (同步，<10ms)
   - 加载 `.env` 文件
   - 设置 Java 环境变量

2. **日志系统初始化** (同步，<50ms)
   - `initLogger()` - 创建日志目录和文件

3. **i18n 初始化** (同步，<50ms)
   - `initI18n()` - 加载语言文件

4. **检查已有实例** (异步，最多 500ms)
   - `attemptDelegationToRunningInstance()`
   - HTTP 请求到 `127.0.0.1:52521`
   - 超时时间 500ms
   - **潜在瓶颈**: 如果端口被占用但服务未响应，会等待超时

**总耗时**: 0.5-1s

---

### 阶段 2: app.whenReady()

**文件**: `meta-doc/src/main/index.ts:464`

**关键操作**:
1. **数据库迁移** (同步，0.2-1s)
   ```typescript
   const { ensureInitialized } = await import('./database/knowledge-db');
   ensureInitialized();
   ```
   - 检查表是否存在
   - 执行迁移脚本
   - 加载 sqlite-vec 扩展（可能耗时）
   - **潜在瓶颈**: 首次启动或数据库迁移时可能较慢

2. **创建主窗口** (同步，<100ms)
   - `createWindow()` - 创建 BrowserWindow 实例

3. **注册 IPC 处理器** (同步，<50ms)
   - `mainCalls()` - 注册所有 IPC 事件处理器

**总耗时**: 0.3-1.5s

---

### 阶段 3: Express 服务器启动

**文件**: `meta-doc/src/main/express-server.ts:117`

**关键操作**:
1. **设置中间件** (同步，<50ms)
   - body-parser 配置
   - CORS 配置
   - 静态文件服务配置

2. **注册 API 路由** (异步，0.1-0.5s)
   ```typescript
   setupAPIs().catch(...)  // 异步执行，不阻塞
   ```
   - 设置运行时 API
   - 设置图片 API
   - 设置知识库 API（可能涉及数据库查询）
   - **潜在瓶颈**: `setupKnowledgeAPI()` 会调用 `refreshKnowledgeItems()`，可能涉及文件系统扫描

3. **启动 HTTP 服务器** (同步，<50ms)
   - 监听端口 52521

**总耗时**: 0.1-0.5s（大部分是异步，不阻塞）

---

### 阶段 4: Vue 应用初始化

**文件**: `meta-doc/src/renderer/src/main.js`

**关键操作**:
1. **模块导入** (同步，0.3-1s)
   - mxgraph 初始化（必须在最前）
   - Vue, Pinia, Router
   - ElementPlus（大型组件库）
   - TDesign（大型组件库）
   - Monaco Editor（大型编辑器）
   - 各种工具库和组件
   - **潜在瓶颈**: 大量依赖导入，特别是大型 UI 库

2. **应用创建和挂载** (同步，<100ms)
   - 创建 Vue 应用实例
   - 注册全局组件
   - 挂载到 DOM

**总耗时**: 0.4-1.5s

---

### 阶段 5: App.vue onMounted

**文件**: `meta-doc/src/renderer/src/App.vue:202`

**关键操作**（串行执行）:
1. **初始化全局事件监听器** (同步，<50ms)
   - `initGlobalEventListeners()`

2. **初始化 Monaco 环境** (同步，<100ms)
   - `initMonacoEnvironment()` - Worker 配置

3. **AIChat 会话迁移** (异步，0.1-1s)
   - `autoMigrateAIChatSessions()` - 从 localStorage 迁移到 SQLite
   - **潜在瓶颈**: 如果历史会话很多，可能较慢

4. **加载关键设置** (异步，0.2-0.5s)
   ```typescript
   await initCriticalSettings()
   ```
   - 通过 IPC 调用 `get-setting`（串行调用多个设置）
   - **潜在瓶颈**: IPC 通信延迟，串行执行

5. **应用主题** (异步，<100ms)
   - `applyTheme()` - 可能需要 IPC 调用获取系统主题

6. **自动打开文档** (异步，0.1-0.3s)
   - `autoOpenDoc()` - 读取最近文档列表

7. **加载非关键设置** (异步，不阻塞)
   - `initNonCriticalSettings()` - 后台异步加载

**总耗时**: 0.5-2s（串行部分）

---

### 阶段 6: 窗口 ready-to-show

**文件**: `meta-doc/src/main/index.ts:131`

**关键操作**:
1. **显示窗口** (同步，<50ms)
   - `mainWindow?.show()`

2. **后台初始化工具服务** (异步，0.5-3s)
   ```typescript
   await initializeUtils()
   ```
   - 初始化 RAG 服务
   - 加载向量数据库
   - 加载嵌入缓存
   - **潜在瓶颈**: 如果知识库文件很多，加载可能很慢

3. **预加载辅助窗口** (异步，0.5-2s)
   - `preloadAuxiliaryWindows()` - 串行预加载多个窗口
   - **潜在瓶颈**: 串行执行，每个窗口都需要创建和加载

**总耗时**: 窗口显示立即完成，后台任务 1-5s

## 性能瓶颈识别

### 🔴 高优先级瓶颈

1. **数据库迁移阻塞启动** (0.2-1s)
   - 位置: `app.whenReady()` 中
   - 影响: 阻塞窗口创建
   - 优化: 可以延迟到窗口显示后

2. **Vue 依赖导入耗时** (0.3-1s)
   - 位置: `main.js` 模块导入
   - 影响: 阻塞渲染进程初始化
   - 优化: 代码分割、懒加载

3. **关键设置串行加载** (0.2-0.5s)
   - 位置: `App.vue onMounted` 中 `initCriticalSettings()`
   - 影响: 阻塞主题应用
   - 优化: 批量 IPC 调用或并行化

4. **工具服务初始化阻塞** (0.5-3s)
   - 位置: `ready-to-show` 中 `initializeUtils()`
   - 影响: 虽然异步，但可能影响首次使用
   - 优化: 进一步延迟或按需加载

### 🟡 中优先级瓶颈

5. **检查已有实例超时** (最多 500ms)
   - 位置: 主进程启动时
   - 影响: 如果端口被占用但服务未响应
   - 优化: 减少超时时间或优化检测逻辑

6. **辅助窗口预加载** (0.5-2s)
   - 位置: `ready-to-show` 中
   - 影响: 后台任务，但占用资源
   - 优化: 按需加载或并行化

7. **AIChat 会话迁移** (0.1-1s)
   - 位置: `App.vue onMounted`
   - 影响: 首次启动可能较慢
   - 优化: 延迟到后台执行

### 🟢 低优先级瓶颈

8. **Express API 路由设置** (0.1-0.5s)
   - 位置: Express 服务器启动
   - 影响: 异步执行，不阻塞
   - 优化: 可以进一步优化

9. **Monaco 环境初始化** (0.1s)
   - 位置: `App.vue onMounted`
   - 影响: 较小
   - 优化: 可以延迟到需要时

## 总启动时间估算

### 阻塞时间（用户感知的启动时间）
- 主进程初始化: 0.5-1s
- app.whenReady: 0.3-1.5s
- 窗口创建: 0.1s
- Express 启动: 0.1s（异步）
- Vue 初始化: 0.4-1.5s
- App.vue onMounted（关键部分）: 0.5-2s

**总计阻塞时间: 1.9-6.1秒**

### 后台任务（不阻塞显示）
- 工具服务初始化: 0.5-3s
- 辅助窗口预加载: 0.5-2s
- 非关键设置加载: 0.1-0.5s

**总计后台时间: 1.1-5.5秒**

## 优化建议优先级

### 立即优化（可减少 1-2 秒）

1. **延迟数据库迁移**
   - 将 `ensureInitialized()` 从 `app.whenReady()` 移到窗口显示后
   - 预期收益: 0.2-1s

2. **优化关键设置加载**
   - 批量 IPC 调用或并行化
   - 预期收益: 0.1-0.3s

3. **延迟工具服务初始化**
   - 移到窗口显示后，或按需加载
   - 预期收益: 0.5-1s（感知上）

### 中期优化（可减少 0.5-1 秒）

4. **代码分割和懒加载**
   - 拆分大型依赖（ElementPlus, TDesign, Monaco）
   - 预期收益: 0.3-0.8s

5. **优化检查已有实例逻辑**
   - 减少超时时间或优化检测
   - 预期收益: 0.2-0.5s

6. **延迟辅助窗口预加载**
   - 按需加载或并行化
   - 预期收益: 0.3-0.8s（资源占用）

### 长期优化（可减少 0.3-0.5 秒）

7. **优化 AIChat 会话迁移**
   - 延迟到后台或分批执行
   - 预期收益: 0.1-0.3s

8. **进一步优化 Express 启动**
   - 优化 API 路由注册
   - 预期收益: 0.1-0.2s

---

## 总结

当前启动时间主要受以下因素影响：
1. **数据库迁移**阻塞窗口创建
2. **大量依赖导入**阻塞渲染进程
3. **串行 IPC 调用**延迟设置加载
4. **工具服务初始化**占用资源

通过优化这些瓶颈，预计可以将启动时间从 **2-6 秒** 降低到 **1-3 秒**，提升用户体验。

## 已实施的优化

### ✅ 优化 1: 延迟数据库迁移
**实施时间**: 2024
**文件**: `meta-doc/src/main/index.ts`

**变更内容**:
- 将数据库迁移从 `app.whenReady()` 移到窗口 `ready-to-show` 事件中
- 数据库迁移现在在窗口显示后异步执行，不再阻塞窗口创建

**预期收益**: 减少阻塞时间 0.2-1 秒

---

### ✅ 优化 2: 批量获取设置
**实施时间**: 2024
**文件**: 
- `meta-doc/src/main/main-calls.ts` - 添加 `get-settings-batch` IPC 处理器
- `meta-doc/src/renderer/src/utils/settings.js` - 使用批量加载

**变更内容**:
- 添加了 `get-settings-batch` IPC 处理器，支持一次性获取多个设置
- `initCriticalSettings()` 现在使用批量加载，将多个 IPC 调用合并为一个

**预期收益**: 减少 IPC 调用延迟 0.1-0.3 秒

---

### ✅ 优化 3: 延迟 AIChat 会话迁移
**实施时间**: 2024
**文件**: `meta-doc/src/renderer/src/App.vue`

**变更内容**:
- AIChat 会话迁移从 `onMounted` 立即执行改为延迟 2 秒执行
- 不阻塞窗口显示和主题应用

**预期收益**: 减少阻塞时间 0.1-1 秒（取决于历史会话数量）

---

### ✅ 优化 4: 优化检查已有实例超时
**实施时间**: 2024
**文件**: `meta-doc/src/main/index.ts`

**变更内容**:
- 将检查已有实例的 HTTP 请求超时时间从 500ms 减少到 200ms
- 加快启动检测速度

**预期收益**: 减少阻塞时间 0.2-0.3 秒（当端口被占用时）
