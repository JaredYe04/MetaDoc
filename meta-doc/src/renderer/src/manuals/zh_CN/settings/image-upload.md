# 上传服务设置

## 概述

上传服务设置允许您配置图片上传的目标服务。MetaDoc支持本地服务和自定义API两种上传方式，您可以根据需求选择合适的服务。

## 上传服务类型

### 服务选择

在图片设置页面，当"插入图片操作"设置为"上传"时，可以选择上传服务：

- **本地服务**：将图片保存到本地目录
- **自定义**：使用自定义API上传图片

### 本地服务

本地服务将图片保存到本地文件系统：

- **优点**：完全本地控制，数据安全
- **缺点**：需要配置本地目录
- **适用场景**：本地使用、数据隐私要求高

### 自定义服务

自定义服务使用外部API上传图片：

- **优点**：可以上传到云存储、图床等
- **缺点**：需要配置API接口
- **适用场景**：需要云存储、图片CDN等

## 本地图片目录配置

### 设置目录

使用本地服务时，需要配置图片保存目录：

1. 在图片设置页面，选择"本地服务"
2. 点击"浏览"按钮选择目录
3. 或直接在输入框中输入目录路径
4. 点击"打开"按钮可以在文件管理器中打开目录

### 目录选择

选择图片目录时：

- **浏览按钮**：打开目录选择对话框
- **路径输入**：直接输入目录路径
- **打开按钮**：在文件管理器中打开已设置的目录

### 默认目录

如果不设置本地图片目录，系统会使用默认目录：

- **Windows**：`%APPDATA%/MetaDoc/images`
- **macOS**：`~/Library/Application Support/MetaDoc/images`
- **Linux**：`~/.config/MetaDoc/images`

### 目录管理

- **查看目录**：点击"打开"按钮查看目录内容
- **更改目录**：点击"浏览"按钮选择新目录
- **目录要求**：确保目录存在且有写入权限

## 自定义上传API配置

### API URL配置

使用自定义服务时，需要配置API地址：

1. 在图片设置页面，选择"自定义"服务
2. 在"自定义上传API URL"输入框中输入API地址
3. 格式示例：`https://api.example.com/upload`

### API方法配置

配置API请求方法：

- **POST**：使用POST方法上传（推荐）
- **PUT**：使用PUT方法上传

大多数API使用POST方法，某些特殊API可能使用PUT方法。

### 字段名配置

配置上传文件的字段名：

- **默认值**：`file`
- **自定义**：根据API要求设置字段名

不同的API可能使用不同的字段名，如`file`、`image`、`upload`等。

### API配置示例

**示例1：标准图床API**
```
API URL: https://api.example.com/upload
方法: POST
字段名: file
```

**示例2：自定义字段名API**
```
API URL: https://api.example.com/image
方法: POST
字段名: image
```

**示例3：PUT方法API**
```
API URL: https://api.example.com/upload
方法: PUT
字段名: file
```

## API响应格式

### 响应要求

自定义API需要返回以下格式的JSON响应：

```json
{
  "success": true,
  "imagePath": "https://example.com/image.png"
}
```

### 响应字段

- **success**：布尔值，表示上传是否成功
- **imagePath**：字符串，返回图片的URL或路径

### 错误处理

如果上传失败，API应返回：

```json
{
  "success": false,
  "message": "错误信息"
}
```

## 配置验证

### 测试配置

配置自定义API后，建议测试配置：

1. 在文档中插入一张图片
2. 查看上传结果
3. 如果失败，检查配置是否正确

### 常见问题

**连接失败**：
- 检查API URL是否正确
- 检查网络连接
- 检查API服务是否正常运行

**上传失败**：
- 检查API方法是否正确
- 检查字段名是否正确
- 检查API响应格式是否符合要求

**权限问题**：
- 检查API是否需要认证
- 检查API Key或Token是否正确

## 本地服务配置

### 目录权限

使用本地服务时，确保目录有写入权限：

- **Windows**：检查文件夹权限设置
- **macOS/Linux**：检查目录权限（chmod）

### 目录结构

本地服务会在指定目录保存图片：

- **文件命名**：使用时间戳+原文件名
- **文件格式**：保持原始格式
- **目录结构**：所有图片保存在同一目录

### 图片访问

本地服务的图片可以通过以下方式访问：

- **HTTP服务**：通过运行时服务器的 `/images/` 路径访问（默认地址由应用配置，如 `http://127.0.0.1:52521/images/`）
- **文件路径**：直接使用文件系统路径

## 最佳实践

1. **本地使用**：本地使用推荐本地服务
2. **云存储**：需要云存储时使用自定义API
3. **目录管理**：定期清理图片目录，避免占用过多空间
4. **API测试**：配置自定义API后先测试
5. **备份策略**：重要图片建议同时备份

## 注意事项

1. **配置生效**：配置更改后，新插入的图片才会使用新配置
2. **API兼容性**：确保自定义API符合响应格式要求
3. **目录权限**：确保本地目录有写入权限
4. **网络连接**：自定义API需要网络连接
5. **存储空间**：本地服务会占用本地存储空间
