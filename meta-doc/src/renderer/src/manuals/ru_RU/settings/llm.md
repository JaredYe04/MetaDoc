# Конфигурация LLM

## Обзор

Конфигурация LLM (большой языковой модели) является основным параметром функций MetaDoc AI. Настроив LLM, вы можете включить интеллектуальные функции, такие как AI-диалог, AI-проверка, AI-автодополнение и другие. MetaDoc поддерживает множество поставщиков услуг LLM, вы можете выбрать подходящую модель в соответствии с вашими потребностями.

-   Если AI-функции не нужны, их можно оставить выключенными для экономии ресурсов

## Настройка температуры LLM

<SettingLlmSection mode="demo" />

### Понимание параметра температуры

Температура (Temperature) — это параметр, контролирующий случайность генерируемого AI текста:

-   **Низкая температура (0-0.5)**: Результаты более детерминированы и последовательны, подходят для сценариев, требующих точных ответов
-   **Средняя температура (0.5-1.0)**: Баланс между креативностью и точностью, подходит для большинства сценариев
-   **Высокая температура (1.0-2.0)**: Результаты более разнообразны и креативны, подходят для творческого письма

### Рекомендации по настройке

-   **Техническая документация**: Рекомендуется 0.3-0.5 для обеспечения точности содержания
-   **Творческое письмо**: Рекомендуется 0.7-1.0 для увеличения разнообразия содержания
-   **Генерация кода**: Рекомендуется 0.2-0.4 для гарантии точности кода
-   **Диалоговое общение**: Рекомендуется 0.7-0.9 для поддержания естественности и плавности диалога

Настройка температуры влияет на все функции, использующие LLM, включая AI-диалог, AI-автодополнение, AI-проверку и другие.

## Автоматическое удаление тегов рассуждений

### Описание функции

Некоторые LLM при генерации контента могут включать процесс рассуждения (thinking process), который обычно помечается специальными тегами. После включения опции "Автоматически удалять теги рассуждений" MetaDoc автоматически отфильтрует эти теги, оставляя только окончательно сгенерированный контент.

**Подходящие сценарии**:

-   Использование LLM, поддерживающих процесс рассуждений (например, некоторые модели с открытым исходным кодом)
-   Желание получить более лаконичный результат
-   Нет необходимости просматривать процесс размышлений AI

**Важные замечания**:

-   Если ваша LLM не поддерживает теги рассуждений, эта опция не окажет влияния
-   В некоторых случаях сохранение процесса рассуждений может помочь понять логику принятия решений AI

## Управление конфигурациями

<SettingLlmSection mode="demo" />

### Поддержка нескольких конфигураций

MetaDoc поддерживает создание нескольких конфигураций LLM, что позволяет использовать разные модели в различных сценариях:

-   **Рабочая конфигурация**: Для повседневной работы, использует стабильные и надежные модели
-   **Экспериментальная конфигурация**: Для тестирования новых моделей или функций
-   **Разные поставщики**: Создание независимых конфигураций для разных служб LLM

### Переключение конфигураций

В списке конфигураций слева на странице настроек LLM вы можете:

1.  **Выбрать конфигурацию**: Нажмите на элемент конфигурации, чтобы переключиться на неё
2.  **Просмотреть информацию о конфигурации**: Имя конфигурации отображается в списке
3.  **Определить текущую конфигурацию**: Используемая в данный момент конфигурация будет выделена

После переключения конфигурации все AI-функции немедленно начнут использовать службу LLM из новой конфигурации.

## Индикация состояния конфигурации

<SettingLlmSection mode="demo" />

### Несохраненные изменения

Когда вы изменили конфигурацию, но ещё не сохранили её, система отобразит подсказку "Несохраненные изменения":

-   Рядом с именем конфигурации будет отображаться предупреждающий значок
-   В строке состояния рабочей области будет отображаться "Есть несохраненные изменения"
-   Для сохранения изменений необходимо нажать кнопку "Сохранить изменения"

### Сохранение изменений

После изменения конфигурации не забудьте нажать кнопку "Сохранить изменения":

1.  Нажмите кнопку "Сохранить изменения" в правом верхнем углу рабочей области
2.  Система сохранит все изменения текущей конфигурации
3.  После успешного сохранения статус обновится на "Все изменения сохранены"

### Отмена изменений

Если вы не хотите сохранять текущие изменения:

1.  Нажмите кнопку "Отменить изменения"
2.  Система вернется к состоянию последнего сохранения
3.  Все несохраненные изменения будут отброшены

## Важные замечания

1.  **Безопасность ключей API**: Бережно храните ваши ключи API, не делитесь ими с другими
2.  **Контроль расходов**: Использование служб LLM может повлечь расходы, следите за объемом использования
3.  **Сетевое подключение**: Для использования внешних API требуется стабильное сетевое подключение
4.  **Резервное копирование конфигураций**: Важные конфигурации рекомендуется экспортировать для резервного копирования, чтобы избежать потери
5.  **Выбор модели**: Разные модели обладают разными возможностями и ограничениями, выбирайте в соответствии с потребностями
