# Управление конфигурациями LLM

## Обзор

Управление конфигурациями LLM позволяет создавать, редактировать, удалять и управлять несколькими конфигурациями LLM. Конфигурации отображаются в виде **карточек сетки**, аналогично клиентам агентов: каждая карточка показывает название и тип конфигурации, клик по ней переключает на её использование, поддерживает проверку подключения прямо на карточке, а через контекстное меню — копирование, редактирование, экспорт и удаление.

## Макет интерфейса

### Сетка и карточки

1.  После включения LLM на странице настроек LLM ниже отобразится **сетка конфигураций**.
2.  Каждая **карточка конфигурации** содержит:
    -   **Первая строка**: Название конфигурации
    -   **Вторая строка**: Тип большой модели (например, OpenAI, Tongyi Qianwen, DeepSeek, Ollama и т.д.)
3.  **Клик по карточке** переключает на эту конфигурацию, текущая используемая конфигурация будет выделена **зелёной рамкой**.
4.  В правом верхнем углу сетки находятся кнопки **«Создать конфигурацию»** и **«Импортировать конфигурацию»**.

## Переключение конфигураций

-   **Кликните на любую карточку** в сетке конфигураций, чтобы переключиться на эту конфигурацию.
-   Текущая конфигурация будет отмечена зелёной рамкой и лёгким выделением, все функции AI немедленно начнут использовать LLM-сервис этой конфигурации.

## Проверка подключения

-   Справа на каждой карточке есть кнопка **«Проверить»**, при нажатии на которую тестируются возможности **потокового ответа на вопросы** и **потокового диалога** для этой конфигурации.
-   Во время теста отображается значок загрузки; при нормальном выводе тест автоматически останавливается и отображается **зелёная галочка**; в случае ошибки запроса отображается **красный крест** и краткое сообщение об ошибке.
-   Независимо от результата, повторное нажатие позволяет запустить тест заново.

## Контекстное меню

**Правый клик** на карточке конфигурации открывает меню, поддерживающее:

-   **Копировать конфигурацию**: Создаёт копию этой конфигурации (новое название содержит «(копия)»).
-   **Редактировать конфигурацию**: Открывает диалоговое окно редактирования для изменения названия, типа и различных параметров. **ОК** сохраняет изменения, **Отмена** — нет.
-   **Экспортировать конфигурацию**: Экспортирует текущую конфигурацию в файл JSON.
-   **Удалить конфигурацию**: Удаляет конфигурацию (**предустановленные конфигурации удалить нельзя**, см. ниже).

## Предустановленные конфигурации

**Предустановленные конфигурации** для следующих типов (например, «Ollama (по умолчанию)», «Tongyi Qianwen (по умолчанию)» и т.д.) **нельзя удалить**, но **можно редактировать** (при редактировании **нельзя изменить тип большой модели**):

-   Tongyi Qianwen, DeepSeek, OpenAI официальный, OpenAI совместимый, Google Gemini, Ollama

Пользовательские конфигурации и конфигурации, полученные копированием, можно удалять как обычно.

## Создание конфигураций

### Новая конфигурация

1.  Нажмите **«Создать конфигурацию»** в правом верхнем углу сетки.
2.  В появившемся окне введите название конфигурации и подтвердите.
3.  Система создаст новую конфигурацию на основе **текущей выбранной конфигурации** и автоматически переключится на неё.

**Примечание**: Когда текущая выбранная конфигурация имеет тип «ручной», кнопка создания новой недоступна.

### Импорт конфигурации

1.  Нажмите **«Импортировать конфигурацию»** в правом верхнем углу сетки.
2.  В открывшемся диалоговом окне выбора файлов выберите один или несколько JSON-файлов конфигурации (поддерживается **пакетный выбор**).
3.  Система прочитает и импортирует их по очереди, импортированные конфигурации будут добавлены в список.

Поддерживается формат JSON с одним объектом конфигурации или массивом конфигураций; при импорте генерируется новый ID, чтобы избежать конфликтов с существующими конфигурациями.

## Редактирование конфигурации

1.  **Правый клик** на карточке конфигурации, выберите **«Редактировать конфигурацию»**.
2.  В диалоговом окне редактирования измените **название конфигурации**, **тип большой модели** (можно изменить для непредустановленных конфигураций) и различные параметры для этого типа (адрес API, ключ, модель и т.д.).
3.  Нажмите **ОК** для сохранения или **Отмена** для отмены изменений. **Нет состояния «не сохранено»**: запись происходит только после подтверждения.

Описание параметров конфигурации для различных типов LLM см. в [[settings.llm-types|Конфигурация типов LLM]].

## Удаление конфигурации

1.  **Правый клик** на карточке конфигурации, выберите **«Удалить конфигурацию»** (для предустановленных конфигураций эта опция не отображается).
2.  Подтвердите удаление в диалоговом окне подтверждения.
3.  Если удаляется текущая используемая конфигурация, система автоматически переключится на другую конфигурацию.

## Экспорт конфигурации

-   **Одиночный экспорт**: Правый клик на карточке → **Экспортировать конфигурацию**, сохраняет текущую конфигурацию в файл JSON.
-   Экспортированный файл можно использовать для резервного копирования или восстановления на другом устройстве/аккаунте через «Импортировать конфигурацию».

## Рекомендации

1.  **Соглашение об именовании**: Используйте понятные названия конфигураций, например, «Работа-Ollama», «Эксперимент-OpenAI».
2.  **Регулярное резервное копирование**: Регулярно экспортируйте важные конфигурации для резервного копирования.
3.  **Сначала проверьте, затем используйте**: Для новых или изменённых конфигураций используйте кнопку «Проверить» на карточке, чтобы убедиться в подключении.
4.  **Очистка неиспользуемых конфигураций**: Регулярно удаляйте конфигурации, которые больше не используются, чтобы поддерживать список в порядке.
