# LLM配置

## 概述

LLM（大语言模型）配置是 MetaDoc AI 功能的核心设置。**通过 Steam 获取的正式版本**与早期测试渠道在默认推荐路径上有所不同：

- **Steam 渠道（推荐）**：使用我们托管的、**基于 Cloudflare 的 LLM API 代理服务**（界面中显示为 **「MetaDoc 官方云（Steam）」**），通过 **Steam 充值** 获得 **credits** 额度后使用 AI，一般**无需**自行申请第三方 API Key。
- **自备 API（BYOK）**：属于 **开发人员选项**：仅在 **LLM 配置** 中开启 **实验性选项** 并启用 **「启用实验性连接」** 后，才会显示与原先测试渠道类似的「多配置 + 自定义 API」链路；详见下文「开发人员选项与自备 API」及链接文档。

下文先说明 **Steam 官方云** 的充值、余额与模型切换，再说明 **实验性选项** 与 **完整界面功能**（启用 LLM、温度、配置网格等）。

---

## Steam 渠道：MetaDoc 官方云与额度（推荐）

适用于通过 **Steam** 安装、运行的 MetaDoc。

### 使用方式概要

1. **打开 LLM 设置**：**设置** → **LLM 设置**（或通过顶部菜单进入 **设置**）。
2. **找到「MetaDoc 官方云（Steam）」**：在该区域可查看 **剩余额度（credits）**、选择 **模型**、发起 **充值**，以及使用 **刷新** 同步云端余额。
3. **充值**：点击 **充值**，按 Steam **应用内购（微交易）** 流程购买 credits；请从 Steam 库正常启动本应用，并确保 Steam 客户端可用（若无法发起购买，界面会提示检查 Steam / Greenworks 等条件）。
4. **查看余额**：页面内展示当前 **credits**；也可通过系统托盘中的 **Steam** 入口查看 **账户 Credits 余额** 并发起充值（与界面文案一致）。
5. **计价**：不同 **模型** 的消耗规则可能不同，扣费会按各模型对应的 credits 计价从余额中扣除；具体以应用内展示为准。
6. **随时切换模型**：在官方云区域选择不同 **模型** 后，**AI 对话、补全、校对、Agent 等** 功能会按当前所选模型调用服务；您可在使用过程中按需切换。

## 开发人员选项：实验性连接与自备 API（BYOK）

在 **Steam 渠道** 中，原先面向测试渠道的 **「自己配置 LLM API」**（BYOK）能力被放在 **开发人员选项** 中：需先在 **LLM 配置** 页面展开 **「实验性选项」**，再开启 **「启用实验性连接」**（开启前会有确认提示）。开启后，才会显示 **多配置卡片、API 地址、API Key、多提供商类型** 等与历史版本一致的配置界面。

**重要**：

- 该选项为 **实验性质**，可能与正式、简化的 Steam 官方云体验 **不一致**，并可能导致 **直接向第三方服务商付费**、网络与合规等额外风险。
- **因自备 API 产生的密钥保管、费用、可用性及与第三方服务条款相关的问题，均由您自行承担**；MetaDoc 仅提供客户端侧的配置与连接能力。

<SettingLlmSection mode="demo" />

### 理解温度参数

温度（Temperature）是控制AI生成文本随机性的参数：

- **低温度（0-0.5）**：生成结果更加确定性和一致，适合需要准确答案的场景
- **中等温度（0.5-1.0）**：平衡创造性和准确性，适合大多数场景
- **高温度（1.0-2.0）**：生成结果更加多样和创造性，适合创意写作

### 设置建议

- **技术文档**：推荐0.3-0.5，确保内容准确
- **创意写作**：推荐0.7-1.0，增加内容多样性
- **代码生成**：推荐0.2-0.4，保证代码准确性
- **对话交流**：推荐0.7-0.9，保持对话自然流畅

温度设置会影响所有使用LLM的功能，包括AI对话、AI补全、AI校对等。

## 自动移除推理标签

### 功能说明

某些LLM在生成内容时可能会包含推理过程（thinking process），这些内容通常以特殊标签标记。启用"自动移除推理标签"后，MetaDoc会自动过滤掉这些标签，只保留最终生成的内容。

**适用场景**：

- 使用支持推理过程的LLM（如某些开源模型）
- 希望输出结果更加简洁
- 不需要查看AI的思考过程

**注意事项**：

- 如果您的LLM不支持推理标签，此选项不会产生影响
- 某些情况下，保留推理过程可能有助于理解AI的决策逻辑

## 配置管理

<SettingLlmSection mode="demo" />

### 多配置支持

MetaDoc支持创建多个LLM配置，方便您在不同场景下使用不同的模型：

- **工作配置**：用于日常工作，使用稳定可靠的模型
- **实验配置**：用于测试新模型或新功能
- **不同提供商**：为不同的LLM服务创建独立配置

### 切换配置

在LLM设置页面的配置网格中，您可以：

1. **选择配置**：点击任意配置卡片即可切换到该配置
2. **查看配置信息**：每张卡片显示配置名称与类型
3. **识别当前配置**：当前使用的配置卡片会以绿色边框高亮

切换配置后，所有AI功能会立即使用新配置的LLM服务。编辑配置需通过卡片右键菜单中的「编辑配置」打开对话框，在对话框中修改后点击「确定」保存，点击「取消」则不保存；界面不再区分「未保存」状态。

## 注意事项

1. **API密钥安全**：请妥善保管您的API密钥，不要分享给他人（仅在使用自备 API / 实验性连接时适用）
2. **费用控制**：使用LLM服务可能会产生费用，请注意使用量；Steam 官方云以 credits 为准，自备 API 以第三方服务商计费为准
3. **网络连接**：使用外部API需要稳定的网络连接
4. **配置备份**：重要配置建议导出备份，避免丢失
5. **模型选择**：不同模型有不同的能力和限制，请根据需求选择
