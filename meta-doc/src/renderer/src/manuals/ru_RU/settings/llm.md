# Конфигурация LLM

## Обзор

Конфигурация LLM (большой языковой модели) является основным параметром функций MetaDoc AI. Настроив LLM, вы можете включить интеллектуальные функции, такие как AI-диалог, AI-проверка, AI-автодополнение и другие. MetaDoc поддерживает множество поставщиков услуг LLM, вы можете выбрать подходящую модель в соответствии с вашими потребностями.

### Включение AI-функций

На странице настроек LLM сначала необходимо включить функцию LLM:

1. Найдите переключатель "Включить LLM"
2. Переведите переключатель в состояние "Включено"
3. Система автоматически загрузит конфигурацию LLM по умолчанию

В демонстрационном режиме вы можете интерактивно просматривать макет интерфейса, но изменения не будут фактически сохранены.

После включения LLM вы сможете использовать следующие AI-функции:

- AI-диалог
- AI-проверка
- AI-автодополнение
- Функции AI-помощника
- Фреймворк Agent

**Важные замечания**:

- После включения LLM некоторые функции могут вызывать API, что приведёт к расходам
- Рекомендуется сначала настроить службу LLM, а затем включать её
- Если AI-функции не нужны, их можно оставить выключенными для экономии ресурсов

### Понимание параметра температуры

Температура (Temperature) — это параметр, контролирующий случайность генерируемого AI текста:

- **Низкая температура (0-0.5)**: Результаты более детерминированы и последовательны, подходят для сценариев, требующих точных ответов
- **Средняя температура (0.5-1.0)**: Баланс между креативностью и точностью, подходит для большинства сценариев
- **Высокая температура (1.0-2.0)**: Результаты более разнообразны и креативны, подходят для творческого письма

### Рекомендации по настройке

- **Техническая документация**: Рекомендуется 0.3-0.5 для обеспечения точности содержания
- **Творческое письмо**: Рекомендуется 0.7-1.0 для увеличения разнообразия содержания
- **Генерация кода**: Рекомендуется 0.2-0.4 для гарантии точности кода
- **Диалоговое общение**: Рекомендуется 0.7-0.9 для поддержания естественности и плавности диалога

Настройка температуры влияет на все функции, использующие LLM, включая AI-диалог, AI-дополнение, AI-проверку и т.д.

## Автоматическое удаление тегов рассуждений

### Описание функции

Некоторые LLM при генерации контента могут включать процесс рассуждения (thinking process), этот контент обычно помечен специальными тегами. После включения опции "Автоматически удалять теги рассуждений" MetaDoc автоматически отфильтрует эти теги, оставляя только окончательно сгенерированный контент.

**Подходящие сценарии**:

- Использование LLM, поддерживающих процесс рассуждений (например, некоторые модели с открытым исходным кодом)
- Желание получить более лаконичный результат
- Не требуется просматривать процесс размышлений AI

**Важные замечания**:

- Если ваша LLM не поддерживает теги рассуждений, эта опция не окажет влияния
- В некоторых случаях сохранение процесса рассуждений может помочь понять логику принятия решений AI

## Управление конфигурациями

<SettingLlmSection mode="demo" />

### Поддержка множественных конфигураций

MetaDoc поддерживает создание нескольких конфигураций LLM, что позволяет вам использовать разные модели в различных сценариях:

- **Рабочая конфигурация**: Для повседневной работы, использует стабильную и надёжную модель
- **Экспериментальная конфигурация**: Для тестирования новых моделей или функций
- **Разные поставщики**: Создание независимых конфигураций для разных служб LLM

### Переключение конфигураций

В сетке конфигураций на странице настроек LLM вы можете:

1. **Выбрать конфигурацию**: Нажмите на любую карточку конфигурации, чтобы переключиться на неё
2. **Просмотреть информацию о конфигурации**: Каждая карточка отображает название и тип конфигурации
3. **Определить текущую конфигурацию**: Карточка используемой в данный момент конфигурации будет выделена зелёной рамкой

После переключения конфигурации все AI-функции немедленно начнут использовать службу LLM из новой конфигурации. Редактирование конфигурации осуществляется через диалоговое окно «Редактировать конфигурацию» в контекстном меню карточки; в диалоговом окне внесите изменения и нажмите «ОК» для сохранения, «Отмена» — для отмены; интерфейс больше не различает состояние «Не сохранено».

## Важные замечания

1. **Безопасность API-ключа**: Бережно храните ваш API-ключ, не делитесь им с другими
2. **Контроль расходов**: Использование служб LLM может привести к расходам, следите за объёмом использования
3. **Сетевое подключение**: Для использования внешних API требуется стабильное интернет-соединение
4. **Резервное копирование конфигураций**: Важные конфигурации рекомендуется экспортировать для резервного копирования во избежание потери
5. **Выбор модели**: Разные модели обладают разными возможностями и ограничениями, выбирайте в соответствии с потребностями
