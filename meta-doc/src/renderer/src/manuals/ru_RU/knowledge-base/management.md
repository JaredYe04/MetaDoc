# Управление базой знаний

## Обзор

<KnowledgeBase mode="demo" />

Управление базой знаний — это основная функция системы MetaDoc RAG (Retrieval-Augmented Generation), которая позволяет добавлять документы в базу знаний и предоставлять контекстную информацию для диалогов с ИИ посредством векторного поиска. База знаний помогает ИИ лучше понимать содержание ваших документов и давать более точные ответы.

## Включение базы знаний

### Включение функции базы знаний

На странице настроек базы знаний сначала необходимо включить функцию базы знаний:

1.  Найдите переключатель "Включить базу знаний"
2.  Переведите переключатель в состояние "Включено"
3.  Настройте соответствующие параметры базы знаний

Вы можете получить доступ к управлению базой знаний через верхнюю панель меню:

Интерфейс управления базой знаний предоставляет следующие функции:

-   **Добавление документов**: Поддержка различных форматов, таких как PDF, Word, изображения, Markdown и другие

**Важные замечания**:

-   После включения базы знаний функции ИИ будут выполнять поиск по её содержимому, что может повлиять на скорость отклика
-   База знаний начнет работать только после добавления файлов
-   Рекомендуется включить базу знаний после добавления файлов

## Настройка порога уверенности

### Понимание порога уверенности

Порог уверенности (Score Threshold) управляет критериями фильтрации результатов поиска в базе знаний:

-   **Низкий порог (0.1-0.3)**: Возвращает больше результатов, но может включать нерелевантный контент
-   **Средний порог (0.4-0.6)**: Баланс между релевантностью и количеством, рекомендуется к использованию
-   **Высокий порог (0.7-0.9)**: Возвращает только высокорелевантные результаты, но может упустить связанную информацию

### Рекомендации по настройке

-   **Общие сценарии**: Рекомендуется 0.5 для баланса точности и охвата
-   **Высокие требования к точности**: Рекомендуется 0.7-0.8 для гарантии высокой релевантности результатов
-   **Исследовательский поиск**: Рекомендуется 0.3-0.4 для получения большего количества связанной информации

Настройка порога влияет на все функции ИИ, использующие базу знаний, включая диалог с ИИ, автодополнение ИИ и другие.

## Управление файлами базы знаний

### Добавление файлов в базу знаний

1.  На странице управления базой знаний нажмите кнопку "Добавить файл"
2.  Выберите файл для добавления (поддерживаются различные форматы)
3.  Система автоматически обработает файл:
    -   Преобразует файл в текст
    -   Разделит текст на блоки
    -   Сгенерирует векторные эмбеддинги
    -   Сохранит в базу знаний

**Поддерживаемые форматы файлов**:

-   Markdown (.md)
-   LaTeX (.tex)
-   PDF (.pdf)
-   Word (.docx)
-   Изображения (.png, .jpg и др., распознаются через OCR)
-   Простой текст (.txt)

### Управление списком файлов

<KnowledgeBase mode="demo" />

На странице управления базой знаний отображаются все добавленные файлы:

-   **Имя файла**: Отображает название файла
-   **Статус**: Показывает, включен ли файл
-   **Количество блоков**: Показывает, на сколько блоков разделен файл
-   **Количество векторов**: Показывает количество сгенерированных векторов
-   **Действия**: Предоставляет операции по управлению файлами

### Включение/отключение файлов

Вы можете временно отключить файл, не удаляя его:

1.  Найдите нужный файл в списке файлов
2.  Нажмите кнопку "Включить" или "Отключить"
3.  После отключения файл не будет участвовать в поиске, но его данные сохранятся

**Сценарии использования**:

-   Временное исключение определенных файлов
-   Тестирование эффективности различных комбинаций файлов
-   Сохранение файла, но временное неиспользование

### Удаление файлов

1.  Найдите файл для удаления в списке файлов
2.  Нажмите кнопку "Удалить"
3.  Подтвердите операцию удаления

Удаление файла приведет к:

-   Удалению записи о файле
-   Удалению всех связанных блоков данных
-   Удалению всех связанных векторов
-   Невозможности восстановления операции

**Важные замечания**:

-   Операция удаления необратима, выполняйте ее осторожно
-   Удаление больших файлов может занять некоторое время
-   После удаления файл нужно добавить заново для восстановления

### Переименование файлов

1.  Найдите файл для переименования в списке файлов
2.  Нажмите кнопку "Переименовать"
3.  Введите новое имя файла
4.  Подтвердите переименование

Переименование изменяет только отображаемое имя и не влияет на содержимое файла и векторные данные.

### Предварительный просмотр файлов

Вы можете просмотреть содержимое файлов в базе знаний:

1.  Найдите файл для предварительного просмотра в списке файлов
2.  Нажмите кнопку "Предпросмотр"
3.  Просмотрите текстовое содержимое файла

Функция предварительного просмотра помогает вам:

-   Убедиться в правильности содержимого файла
-   Проверить, правильно ли обработан файл
-   Понять текстовую структуру файла

### Скачивание файлов

Вы можете скачать файлы из базы знаний:

1.  Найдите файл для скачивания в списке файлов
2.  Нажмите кнопку "Скачать"
3.  Выберите место сохранения

Скачанный файл является копией исходного файла и может использоваться для резервного копирования или обмена.

## Реконструкция векторов

### Реконструкция векторов

Если возникли проблемы с векторными данными файла или была обновлена модель эмбеддинга, можно выполнить реконструкцию векторов:

1.  Найдите файл для реконструкции в списке файлов
2.  Нажмите кнопку "Реконструировать векторы"
3.  Дождитесь завершения реконструкции

Реконструкция векторов выполнит:

-   Повторную обработку текста файла
-   Повторную генерацию векторных эмбеддингов
-   Обновление векторного индекса

**Сценарии использования**:

-   Смена модели эмбеддинга
-   Повреждение векторных данных
-   Необходимость обновления векторного представления

### Реконструкция всех векторов

Если необходимо реконструировать векторы для всех файлов:

1.  Нажмите кнопку "Реконструировать все векторы"
2.  Подтвердите операцию
3.  Дождитесь завершения реконструкции всех файлов

Реконструкция всех векторов может занять значительное время, особенно при большом количестве файлов.

## Тестирование поиска в базе знаний

### Тестирование функции поиска

Вы можете протестировать функцию поиска на странице управления базой знаний:

1.  Введите поисковый запрос в поле поиска
2.  Нажмите кнопку "Поиск"
3.  Просмотрите результаты поиска

Результаты поиска будут отображать:

-   Соответствующие фрагменты текста
-   Оценку сходства (score)
-   Файл-источник
-   Контекстную информацию

### Параметры поиска

При тестировании поиска можно настраивать:

-   **Текст запроса**: Введите содержание для поиска
-   **Количество результатов**: Установите количество возвращаемых результатов
-   **Порог**: Установите минимальный порог сходства

## Очистка базы знаний

### Очистка всех данных

Если необходимо очистить всю базу знаний:

1.  Нажмите кнопку "Очистить базу знаний"
2.  Подтвердите операцию
3.  Дождитесь завершения очистки

Очистка базы знаний выполнит:

-   Удаление всех записей о файлах
-   Удаление всех блоков данных
-   Удаление всех векторов
-   Невозможность восстановления операции

**Важные замечания**:

-   Операция очистки необратима, выполняйте ее осторожно
-   Перед очисткой рекомендуется создать резервную копию важных файлов
-   После очистки файлы необходимо добавить заново

## Рекомендации по использованию

1.  **Организация файлов**: Организуйте файлы по темам или проектам для удобства управления
2.  **Регулярное обновление**: После обновления содержимого файлов своевременно реконструируйте векторы
3.  **Настройка порога**: Настраивайте порог уверенности в зависимости от фактической эффективности использования
4.  **Очистка файлов**: Регулярно удаляйте ненужные файлы для поддержания порядка в базе знаний
5.  **Резервное копирование важных файлов**: Важные файлы рекомендуется сначала скопировать, а затем добавлять в базу знаний

## Важные замечания

1.  **Размер файлов**: Обработка больших файлов занимает больше времени, пожалуйста, наберитесь терпения
2.  **Дисковое пространство**: База знаний занимает определенное место на диске
3.  **Время обработки**: Добавление файлов и обработка векторов требуют времени, не прерывайте процесс
4.  **Формат файлов**: Убедитесь в правильности формата файлов, иначе они могут не обработаться
5.  **Сетевое соединение**: Для генерации векторов в режиме API требуется сетевое соединение
