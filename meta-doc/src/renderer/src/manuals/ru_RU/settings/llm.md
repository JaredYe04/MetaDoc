# Настройка LLM

## Обзор

Конфигурация LLM (большой языковой модели) — основа ИИ-функций MetaDoc. **Версии, распространяемые через Steam**, отличаются от старых тестовых каналов рекомендуемым путём:

- **Steam (рекомендуется)**: используйте **LLM API-прокси на базе Cloudflare**, который ведёт MetaDoc (в приложении — **«MetaDoc Cloud (Steam)»** и т. п.). Покупайте **кредиты (credits)** через **пополнение в Steam** и пользуйтесь ИИ — обычно **без** собственных ключей сторонних API.
- **Свой API (BYOK)**: **экспериментальный / для разработчиков** сценарий. Только после раздела **Экспериментальные опции** на странице **LLM** и включения **«Включить экспериментальное подключение»** появляется привычный интерфейс **нескольких профилей и своего API**. Подробности ниже и в связанных статьях.

Сначала: **Steam / MetaDoc Cloud** (баланс, пополнение, смена модели), затем **экспериментальные опции**, затем **остальной интерфейс** (включение LLM, температура, сетка карточек и т. д.).

## Steam: MetaDoc Cloud и кредиты (рекомендуется)

Для MetaDoc, установленного и запущенного через **Steam**.

### Кратко

1. **Открыть настройки LLM**: **Настройки** → **LLM** (или **Настройки** в меню).
2. Найти **«MetaDoc Cloud (Steam)»**: **баланс (credits)**, выбор **модели**, **пополнение**, **обновление** баланса на сервере.
3. **Пополнение**: через **добавление кредитов** — **встроенная покупка Steam**. Запускайте приложение из библиотеки Steam, клиент Steam должен быть доступен (при сбое интерфейс подскажет про Steam/Greenworks).
4. **Баланс**: кредиты отображаются на странице; через значок **Steam** в трее — **баланс credits** и пополнение.
5. **Ценообразование**: у разных **моделей** могут быть разные правила списания — смотрите приложение.
6. **Смена модели**: после выбора в MetaDoc Cloud **чат ИИ, дополнение, корректура, Agent** используют эту модель; можно менять в любой момент.

### Примечания

- Конечная точка и прокси обслуживаются MetaDoc: для этого режима **не нужно** вводить URL/ключи OpenAI или DeepSeek (в отличие от BYOK).
- Недостаточный баланс или сеть: ИИ может быть недоступен — **пополните** или **обновите баланс**.

## Параметры разработчика: экспериментальное подключение и BYOK (свой API)

В **Steam**-сборках старый сценарий **«настроить свой LLM API»** — **экспериментальный**: откройте **Экспериментальные опции** на странице **LLM**, включите **«Включить экспериментальное подключение»** (с подтверждением). Только тогда появляются **карточки конфигураций, базовый URL API, ключ API, типы провайдеров**, как в тестовых сборках.

**Важно**:

- Режим **экспериментальный**, опыт может **отличаться** от стандартного; возможны **прямые платежи** сторонним провайдерам и иные риски.
- **Вы несёте полную ответственность** за ключи, оплату, доступность и условия третьих сторон. MetaDoc даёт только клиентскую настройку.

### Интерфейс

<SettingLlmSection mode="demo" />

<MenuItemsDemo mode="demo" :items='[{"id": "settings"}]' />

- **Глобальные настройки**: переключатель LLM, температура, удаление тегов рассуждений, разрешение терминала по умолчанию и т. д.
- **Сетка карточек**: имя и тип; клик переключает активную конфигурацию (зелёная рамка)
- **Действия**: проверка потока; контекстное меню: копировать, править, экспорт, удалить
- **Сверху справа**: новая конфигурация, импорт из файла

В демо-режиме изменения не сохраняются.

После включения: чат ИИ, корректура, дополнение, ассистент, Agent.

**Замечание**: вызовы API могут стоить денег.

## Важно

1. **Ключи API**: только для BYOK — не передавайте другим
2. **Стоимость**: MetaDoc Cloud — **credits**; BYOK — биллинг провайдера
3. **Сеть**: стабильное соединение с внешними API
4. **Резервное копирование**: экспортируйте важные конфигурации
5. **Модели**: разные возможности и ограничения
