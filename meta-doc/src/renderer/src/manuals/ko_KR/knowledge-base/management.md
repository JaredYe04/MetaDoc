# 지식베이스 관리

## 개요

<KnowledgeBase mode="demo" />

지식베이스 관리는 MetaDoc RAG(검색 증강 생성) 시스템의 핵심 기능으로, 문서를 지식베이스에 추가하여 벡터 검색을 통해 AI 대화에 컨텍스트 정보를 제공할 수 있게 합니다. 지식베이스는 AI가 귀하의 문서 내용을 더 잘 이해하고 더 정확한 답변을 제공하도록 돕습니다.

## 지식베이스 활성화

### 지식베이스 기능 켜기

지식베이스 설정 페이지에서 먼저 지식베이스 기능을 활성화해야 합니다:

1. "지식베이스 활성화" 스위치를 찾습니다.
2. 스위치를 "활성화" 상태로 전환합니다.
3. 지식베이스 관련 매개변수를 구성합니다.

상단 메뉴 바를 통해 지식베이스 관리에 접근할 수 있습니다:

### 지식베이스 설정

<SettingKnowledgeBaseSection mode="demo" />

지식베이스를 활성화하기 전에 설정 페이지에서 관련 매개변수를 구성할 수 있습니다:

위 그림은 지식베이스 설정 인터페이스의 주요 옵션을 보여줍니다:

- **지식베이스 활성화**: 지식베이스 기능을 켜거나 끕니다.
- **임베딩 모드**: 클라우드 처리 또는 로컬 처리(개발 중)를 선택합니다.
- **신뢰도 임계값**: 검색 결과의 관련성 필터링을 제어합니다.
- **최대 검색 수**: 각 검색 시 반환되는 최대 결과 수를 제한합니다.

### 지식베이스 관리 인터페이스

<KnowledgeBase mode="demo" />

지식베이스를 활성화한 후, 지식베이스 관리 인터페이스에서 문서를 추가하고 관리할 수 있습니다:

지식베이스 관리 인터페이스는 다음 기능을 제공합니다:

- **문서 목록**: 지식베이스에 추가된 모든 문서를 확인합니다.
- **문서 추가**: PDF, Word, 이미지, Markdown 등 다양한 형식을 지원합니다.
- **처리 상태**: 문서 처리 진행 상황을 실시간으로 표시합니다.
- **검색 테스트**: 지식베이스의 검색 효과를 테스트합니다.

지식베이스를 활성화하면 AI 기능(예: AI 대화, AI 완성)이 자동으로 지식베이스의 정보를 사용하여 답변 품질을 향상시킵니다.

**주의사항**:

- 지식베이스를 활성화하면 AI 기능이 지식베이스 내용을 검색하므로 응답 속도에 영향을 줄 수 있습니다.
- 지식베이스는 파일을 먼저 추가해야 효과가 있습니다.
- 파일을 추가한 후 지식베이스를 활성화하는 것이 좋습니다.

## 신뢰도 임계값 설정

### 신뢰도 임계값 이해하기

신뢰도 임계값(Score Threshold)은 지식베이스 검색 결과의 필터링 기준을 제어합니다:

- **낮은 임계값 (0.1-0.3)**: 더 많은 결과를 반환하지만 관련 없는 내용이 포함될 수 있습니다.
- **중간 임계값 (0.4-0.6)**: 관련성과 수량의 균형을 맞추며, 권장 설정입니다.
- **높은 임계값 (0.7-0.9)**: 매우 관련성 높은 결과만 반환하지만 관련 정보를 놓칠 수 있습니다.

### 설정 권장사항

- **일반적인 시나리오**: 정확성과 커버리지 균형을 위해 0.5를 권장합니다.
- **높은 정밀도 요구**: 결과의 높은 관련성을 보장하기 위해 0.7-0.8을 권장합니다.
- **탐색적 검색**: 더 많은 관련 정보를 얻기 위해 0.3-0.4를 권장합니다.

임계값 설정은 AI 대화, AI 완성 등 지식베이스를 사용하는 모든 AI 기능에 영향을 미칩니다.

## 지식베이스 파일 관리

### 지식베이스에 파일 추가하기

1. 지식베이스 관리 페이지에서 "파일 추가" 버튼을 클릭합니다.
2. 추가할 파일을 선택합니다(다양한 형식 지원).
3. 시스템이 자동으로 파일을 처리합니다:
   - 파일을 텍스트로 변환
   - 텍스트를 청크로 분할
   - 벡터 임베딩 생성
   - 지식베이스에 저장

**지원하는 파일 형식**:

- Markdown (.md)
- LaTeX (.tex)
- PDF (.pdf)
- Word (.docx)
- 이미지 (.png, .jpg 등, OCR로 인식)
- 일반 텍스트 (.txt)

### 파일 처리 절차

```mermaid

    D --> E[지식베이스 저장]
```

### 파일 목록 관리

<KnowledgeBase mode="demo" />

지식베이스 관리 페이지는 추가된 모든 파일을 표시합니다:

- **파일명**: 파일의 이름을 표시합니다.
- **상태**: 파일이 활성화되었는지 여부를 표시합니다.
- **청크 수**: 파일이 분할된 청크 수를 표시합니다.
- **벡터 수**: 생성된 벡터 수를 표시합니다.
- **작업**: 파일 관리 작업을 제공합니다.

### 파일 활성화/비활성화

파일을 삭제하지 않고 일시적으로 비활성화할 수 있습니다:

1. 파일 목록에서 작업할 파일을 찾습니다.
2. "활성화" 또는 "비활성화" 버튼을 클릭합니다.
3. 비활성화 후, 해당 파일은 검색되지 않지만 데이터는 유지됩니다.

**사용 시나리오**:

- 특정 파일을 일시적으로 제외
- 다른 파일 조합의 효과 테스트
- 파일을 유지하지만 일시적으로 사용하지 않음

### 파일 삭제

1. 파일 목록에서 삭제할 파일을 찾습니다.
2. "삭제" 버튼을 클릭합니다.
3. 삭제 작업을 확인합니다.

파일 삭제 시:

- 파일 기록 삭제
- 모든 관련 데이터 청크 삭제
- 모든 관련 벡터 삭제
- 작업은 복구할 수 없음

**주의사항**:

- 삭제 작업은 복구할 수 없으므로 신중하게 작업하십시오.
- 대용량 파일 삭제에는 시간이 걸릴 수 있습니다.
- 삭제 후 다시 추가해야 복구할 수 있습니다.

### 파일 이름 변경

1. 파일 목록에서 이름을 변경할 파일을 찾습니다.
2. "이름 변경" 버튼을 클릭합니다.
3. 새 파일 이름을 입력합니다.
4. 이름 변경을 확인합니다.

이름 변경은 표시 이름만 변경하며, 파일 내용과 벡터 데이터에는 영향을 미치지 않습니다.

### 파일 미리보기

지식베이스의 파일 내용을 미리 볼 수 있습니다:

1. 파일 목록에서 미리 볼 파일을 찾습니다.
2. "미리보기" 버튼을 클릭합니다.
3. 파일의 텍스트 내용을 확인합니다.

미리보기 기능은 다음을 도와줍니다:

- 파일 내용이 올바른지 확인
- 파일이 올바르게 처리되었는지 확인
- 파일의 텍스트 구조 파악

### 파일 다운로드

지식베이스의 파일을 다운로드할 수 있습니다:

1. 파일 목록에서 다운로드할 파일을 찾습니다.
2. "다운로드" 버튼을 클릭합니다.
3. 저장 위치를 선택합니다.

다운로드된 파일은 원본 파일의 복사본으로, 백업 또는 공유에 사용할 수 있습니다.

## 벡터 재구성

### 벡터 재구성

파일의 벡터 데이터에 문제가 있거나 Embedding 모델을 업데이트한 경우 벡터를 재구성할 수 있습니다:

1. 파일 목록에서 재구성할 파일을 찾습니다.
2. "벡터 재구성" 버튼을 클릭합니다.
3. 재구성이 완료될 때까지 기다립니다.

벡터 재구성 시:

- 파일 텍스트 재처리
- 벡터 임베딩 재생성
- 벡터 인덱스 업데이트

**사용 시나리오**:

- Embedding 모델 변경
- 벡터 데이터 손상
- 벡터 표현 업데이트 필요

### 전체 벡터 재구성

모든 파일의 벡터를 재구성해야 하는 경우:

1. "전체 벡터 재구성" 버튼을 클릭합니다.
2. 작업을 확인합니다.
3. 모든 파일 재구성이 완료될 때까지 기다립니다.

전체 벡터 재구성에는 시간이 오래 걸릴 수 있으며, 특히 파일이 많을 경우 더욱 그렇습니다.

## 지식베이스 검색 테스트

### 검색 기능 테스트하기

지식베이스 관리 페이지에서 검색 기능을 테스트할 수 있습니다:

1. 검색 상자에 쿼리 텍스트를 입력합니다.
2. "검색" 버튼을 클릭합니다.
3. 검색 결과를 확인합니다.

검색 결과는 다음을 표시합니다:

- 일치하는 텍스트 조각
- 유사도 점수
- 출처 파일
- 컨텍스트 정보

### 검색 매개변수

검색 테스트 시 다음을 조정할 수 있습니다:

- **쿼리 텍스트**: 검색할 내용 입력
- **반환 수**: 반환할 결과 수 설정
- **임계값**: 최소 유사도 임계값 설정

## 지식베이스 비우기

### 모든 데이터 비우기

전체 지식베이스를 비워야 하는 경우:

1. "지식베이스 비우기" 버튼을 클릭합니다.
2. 작업을 확인합니다.
3. 비우기 완료를 기다립니다.

지식베이스 비우기 시:

- 모든 파일 기록 삭제
- 모든 데이터 청크 삭제
- 모든 벡터 삭제
- 작업은 복구할 수 없음

**주의사항**:

- 비우기 작업은 복구할 수 없으므로 신중하게 작업하십시오.
- 비우기 전에 중요한 파일을 백업하는 것이 좋습니다.
- 비운 후에는 파일을 다시 추가해야 합니다.

## 모범 사례

1. **파일 구성**: 주제 또는 프로젝트별로 파일을 구성하여 관리하기 쉽게 합니다.
2. **정기적 업데이트**: 파일 내용 업데이트 후, 벡터를 즉시 재구성합니다.
3. **임계값 조정**: 실제 사용 효과에 따라 신뢰도 임계값을 조정합니다.
4. **파일 정리**: 더 이상 필요하지 않은 파일을 정기적으로 삭제하여 지식베이스를 깔끔하게 유지합니다.
5. **중요 파일 백업**: 중요한 파일은 지식베이스에 추가하기 전에 백업하는 것이 좋습니다.

## 주의사항

1. **파일 크기**: 대용량 파일 처리에는 시간이 오래 걸리므로 기다려 주십시오.
2. **저장 공간**: 지식베이스는 일정량의 저장 공간을 차지합니다.
3. **처리 시간**: 파일 추가 및 벡터 처리에는 시간이 필요하며, 중단하지 마십시오.
4. **파일 형식**: 파일 형식이 올바른지 확인하고, 그렇지 않으면 처리되지 않을 수 있습니다.
5. **네트워크 연결**: API 모드로 벡터 생성 시 네트워크 연결이 필요합니다.
