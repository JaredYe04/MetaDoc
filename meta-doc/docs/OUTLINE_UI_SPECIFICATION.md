## 一、左侧 AI 工具栏

### 位置

主编辑区**左上方**，紧邻"大纲"视图切换按钮的**右侧**

### 排列方式

**纵向单列排列**（每行一个按钮）

### 显示逻辑

- **常态**：仅显示 **Icon**（无 Tooltip，纯图标状态）
- **激活态**：鼠标 **hover** 或 **点击选择** 时，按钮**向右展开**（溢出显示），展示 **Icon + 完整文字标签**

### 功能按钮（从上到下）

| #   | 功能           | 图标                           | 说明                                               |
| --- | -------------- | ------------------------------ | -------------------------------------------------- |
| 1   | 生成子章节     | 节点网络图标 (BranchIcon)      | 为当前选中节点 AI 生成子级结构                     |
| 2   | 生成内容       | 铅笔图标 (WriteIcon)           | 为当前节点生成详细文本                             |
| 3   | 批量生成子章节 | 批量组织图标 (MultiBranchIcon) | 批量创建多个子章节                                 |
| 4   | 批量生成内容   | 批量闪电图标 (MultiWriteIcon)  | 批量填充节点内容                                   |
| 5   | 格式化标题     | 文本排版图标 (FormatIcon)      | 自动规范化标题层级 ⚠️ **当前缺失文字标签，需补充** |

### 修复要点

- [x] 纵向排列（垂直方向）
- [x] 一行一个按钮
- [x] 正常情况下不溢出
- [ ] 只显示 icon，hover/点击时显示完整 icon + 文字
- [ ] 按钮向右溢出展开
- [ ] **不需要 Tooltip** - hover/点击直接显示文字
- [ ] "格式化标题"按钮要显示文字（现在缺失）
- [ ] 所有按钮圆角半径稍大一些

## 三、主编辑区（中央画布）

### 3.1 核心结构

**容器特性**：

- `tree-container` 大小**固定**，紧贴父容器边界
- **不参与缩放变换**（关键！）

**节点层级**：

- **根节点**：顶部中央的蓝色胶囊节点（带展开按钮）
- **一级节点**：横向排列的五个蓝色胶囊节点
  - 一、核心概念
  - 二、工作...
  - 三、实践方法论：追求闭环与自动化
  - 四、实施策略：先搭架构，再填功能
  - 总结

### 3.2 节点样式规格（重要）

**圆角**：

- [ ] 胶囊形状
- [ ] **圆角半径较大**（接近全圆角 rounded-full）

**间距**：

- [ ] 同层级节点间保持**较大间隙**（现在太紧凑，节点挨着）
- [ ] 避免视觉拥挤

**展开按钮**（关键细节）：

- [ ] **图标样式**：**无柄箭头**（仅含 `>` 或 `v` 的单一斜线，无横线柄部）
- [ ] **位置**：节点右侧
- [ ] **视觉统一**：与节点文字/背景视觉统一

**AI 工具替换逻辑**：

- [ ] 当用户在 AI 工具栏选择工具时，展开按钮**暂时隐藏**
- [ ] 原位置显示**当前选中 AI 工具的 Icon**
- [ ] 样式保持与展开按钮一致，仅替换图标内容

### 3.3 展开行为

**位置保持**：

- [ ] 点击展开/折叠时，**大纲树图整体位置不应发生位移**（视口稳定）
- [ ] 展开组件显示位置应**精确对齐原父节点的垂直位置**

## 四、底部浮动控制栏

### 位置

编辑区底部**居中悬浮**（fixed positioning）

### 样式

- [ ] 浮动黑色工具条
- [ ] 按钮大小与间距**适中均衡**

### 功能

- **放大（+）** - 调整画布缩放比例
- **缩小（-）** - 调整画布缩放比例
- **重置视图** - 恢复默认缩放与位置
- **适应屏幕** - 自动调整至最佳可视范围

### 缩放逻辑（关键修复）

- [ ] 缩放操作**仅作用于 `tree-container` 内部的大纲树内容**
- [ ] `tree-container` 容器本身**保持固定大小**，不随缩放变换
- [ ] 避免当前直接缩放整个容器的问题
