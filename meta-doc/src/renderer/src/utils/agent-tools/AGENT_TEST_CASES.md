# Agent测试用例

本文档包含一系列Agent测试用例，用于验证各种工具的正确性和工具间的协作能力。

## 基础用例

### 1. 分析文档并概括内容
**描述**：分析当前文档的内容并生成摘要  
**预期工具使用**：
- `outline-tree` 或 `grep` - 了解文档结构
- 可能生成新的内容（通过edit工具）

**测试命令**：
```
分析这篇文档的内容，并用一段话概括主要观点
```

---

### 2. 分析文章内容并生成精美的PlantUML图表
**描述**：分析文章内容，提取关键信息，生成PlantUML图表  
**预期工具使用**：
- `grep` 或 `outline-tree` - 分析文章内容
- `chart-generation` - 生成PlantUML图表（format: pdf for LaTeX, svg/png for Markdown）
- `edit` - 插入图表URL到合适位置

**测试命令**：
```
分析这篇文章的内容，提取其中的系统架构或流程，生成一个精美的PlantUML类图或序列图
```

---

### 3. 把文章中所有的"前端"替换成"前台"
**描述**：全局替换文档中的特定文本  
**预期工具使用**：
- `edit` (findReplace with all: true) - 全局替换

**测试命令**：
```
把文章中所有的"前端"替换成"前台"
```

---

### 4. 在文章的合适位置插入合适的图表
**描述**：根据文档内容，在合适的位置插入相关图表  
**预期工具使用**：
- `grep` 或 `outline-tree` - 定位插入位置
- `chart-generation` - 生成图表

## 测试建议

1. **从简单用例开始**：先测试基础用例（1-4），确保核心功能正常
2. **逐步增加复杂度**：然后测试扩展用例（5-13），验证工具协作
3. **综合测试**：最后测试复杂综合用例（14-25），验证完整工作流
4. **关注错误处理**：注意测试边界情况和错误处理
5. **验证工具组合**：特别关注多个工具组合使用的场景

## 注意事项

- 所有用例应该在实际文档上测试
- 测试时注意观察Agent的工具调用顺序和逻辑
- 验证每个工具的输出是否符合预期
- 检查最终结果是否符合要求
