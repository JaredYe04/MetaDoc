# LLM配置管理

## 概述

LLM配置管理允許您建立、編輯、刪除和管理多個LLM配置。配置以**網格卡片**形式展示，類似代理客戶端：每個卡片顯示配置名稱與類型，點擊即可切換使用，支援在卡片上檢查連通性，透過右鍵選單進行複製、編輯、匯出與刪除。

## 介面佈局

### 網格與卡片

1.  在LLM設定頁面啟用LLM後，下方會顯示**配置網格**。
2.  每個**配置卡片**包含：
    -   **第一行**：配置名稱
    -   **第二行**：大模型類型（如 OpenAI、通義千問、DeepSeek、Ollama 等）
3.  **點擊卡片**即可切換到該配置，目前使用的配置卡片會以**綠色邊框**高亮。
4.  網格右上角有**新建配置**和**匯入配置**按鈕。

## 切換配置

-   在配置網格中**點擊任意卡片**即可切換到該配置。
-   目前配置會以綠色邊框和輕微高亮顯示，所有AI功能將立即使用該配置的LLM服務。

## 檢查連通性

-   每個卡片右側有**「檢查」**按鈕，點擊後會測試該配置的**問答串流**與**對話串流**能力。
-   測試中會顯示載入圖示；若有正常輸出則自動停止並顯示**綠色勾**；若請求錯誤則顯示**紅色叉**及簡短錯誤訊息。
-   無論結果如何，再次點擊均可重新測試。

## 右鍵選單

在配置卡片上**右鍵**可開啟選單，支援：

-   **複製配置**：建立該配置的副本（新名稱帶「(副本)」）。
-   **編輯配置**：開啟編輯對話方塊，修改名稱、類型及各項參數，**確定**即儲存，**取消**則不儲存。
-   **匯出配置**：將目前配置匯出為 JSON 檔案。
-   **刪除配置**：刪除該配置（**預設配置不可刪除**，見下文）。

## 預設配置

以下類型對應的**預設配置**（如「Ollama (預設)」「通義千問 (預設)」等）**不可刪除**，但**允許編輯**（編輯時**不可更改大模型類型**）：

-   通義千問、DeepSeek、OpenAI官方、OpenAI相容、Google Gemini、Ollama

自訂配置及複製得到的配置可以正常刪除。

## 建立配置

### 新建配置

1.  點擊網格右上角**「新建配置」**。
2.  在彈出視窗中輸入配置名稱並確認。
3.  系統會基於**目前選中的配置**建立新配置並自動切換到新配置。

**注意**：目前選中的配置為「手動」類型時，新建按鈕不可用。

### 匯入配置

1.  點擊網格右上角**「匯入配置」**。
2.  在開啟的檔案對話方塊中選擇一個或多個 JSON 配置檔案（支援**批次選擇**）。
3.  系統會逐個讀取並匯入，匯入的配置會追加到清單中。

支援單個配置物件或配置陣列的 JSON 格式；匯入時會產生新 ID，避免與現有配置衝突。

## 編輯配置

1.  在配置卡片上**右鍵**，選擇**「編輯配置」**。
2.  在編輯對話方塊中修改**配置名稱**、**大模型類型**（非預設配置可改）以及該類型下的各項參數（API 位址、金鑰、模型等）。
3.  點擊**確定**儲存，或**取消**放棄修改。**無「未儲存」狀態**：只有確認後才會寫入。

不同LLM類型的配置參數說明見 [[settings.llm-types|LLM類型配置]]。

## 刪除配置

1.  在配置卡片上**右鍵**，選擇**「刪除配置」**（預設配置不顯示該選項）。
2.  在確認對話方塊中確認刪除。
3.  若刪除的是目前使用的配置，系統會自動切換到其他配置。

## 匯出配置

-   **單個匯出**：在卡片上右鍵 → **匯出配置**，將目前配置儲存為 JSON 檔案。
-   匯出的檔案可用於備份或在其他裝置/帳號中透過「匯入配置」恢復。

## 最佳實踐

1.  **命名規範**：使用清晰的配置名稱，如「工作-Ollama」「實驗-OpenAI」。
2.  **定期備份**：重要配置定期匯出備份。
3.  **先檢查後使用**：新配置或修改後可用卡片上的「檢查」確認連通性。
4.  **清理無用配置**：定期刪除不再使用的配置，保持清單整潔。
